  1416. 1416 <li>Changed how cache files are read to prevent PHP errors if the cache file contains an XML tag, which PHP wants to interpret as a short tag.</li>
  1417. 1417 <li>Fixed a bug in a couple of the active record functions (where and orderby).</li>
  1418. 1418 <li>Fixed a bug in the image library when realpath() returns false.</li>
  1419. 1419 <li>Fixed a bug in the Models that was preventing libraries from being used within them.</li>
  1420. 1420 <li>Fixed a bug in the "exact_length" function of the validation class.</li>

  1425. 1425 <h2>Version 1.3</h2>
  1426. 1426 <p>Release Date: April 3, 2006</p>
  1427. 1427 
  1428. 1428 <ul>
  1429. 1429 <li>Added support for <a href="models.html">Models</a>.</li>
  1430. 1430 <li>Redesigned the database libraries to support additional RDBMs (Postgres, MySQLi, etc.).</li>
  1431. 1431 <li>Redesigned the <a href="./database/active_record.html">Active Record class</a> to enable more varied types of queries with simpler syntax, and advanced features like JOINs.</li>
  1432. 1432 <li>Added a feature to the database class that lets you run <a href="./database/call_function.html">custom function calls</a>.</li>
  1433. 1433 <li>Added support for <a href="controllers.html">private functions</a> in your controllers.  Any controller function name that starts with an underscore will not be served by a URI request.</li>
  1434. 1434 <li>Added the ability to pass your own initialization parameters to your <a href="creating_libraries.html">custom core libraries</a> when using $this->load->library()</li>
  1435. 1435 <li>Added support for running standard <a href="urls.html">query string URLs</a>.  These can be optionally enabled in your config file.</li>
  1436. 1436 <li>Added the ability to  <a href="urls.html">specify a "suffix"</a>, which will be appended to your URLs.  For example, you could add .html to your URLs, making them appear static. This feature is enabled in your config file.</li>
  1437. 1437 <li>Added a new error template for use with native PHP errors.</li>
  1438. 1438 <li>Added "alternator" function in the <a href="./helpers/string_helper.html">string helpers</a>.</li>
  1439. 1439 <li>Removed slashing from the input class. After much debate we decided to kill this feature.</li>
  1440. 1440 <li>Change the commenting style in the scripts to the PEAR standard so that IDEs and tools like phpDocumenter can harvest the comments.</li>
  1441. 1441 <li>Added better class and function name-spacing to avoid collisions with user developed classes. All CodeIgniter classes are now prefixed with <dfn>CI_</dfn> and
  1442. 1442 all controller methods are prefixed with <dfn>_ci</dfn> to avoid controller collisions. A list of reserved function names can be <a href="controllers.html">found here</a>.</li>
  1443. 1443 <li>Redesigned how the "CI" super object is referenced, depending on whether PHP 4 or 5 is being run, since PHP 5 allows a more graceful way to manage objects that utilizes a bit less resources.</li>
  1444. 1444 <li>Deprecated: <var>$this->db->use_table()</var> has been deprecated. Please read the <a href="./database/active_record.html">Active Record</a> page for information.</li>
  1445. 1445 <li>Deprecated: <var>$this->db->smart_escape_str()</var> has been deprecated. Please use this instead: <var>$this->db->escape()</var></li>
  1446. 1446 <li>Fixed a bug in the exception handler which was preventing some PHP errors from showing up.</li>
  1447. 1447 <li>Fixed a typo in the URI class.  $this->total_segment() should be plural: $this->total_segments()</li>
  1448. 1448 <li>Fixed some typos in the default calendar template</li>
  1449. 1449 <li>Fixed some typos in the user guide</li>
