  57.  57 <h1>Active Record Class</h1>
  58.  58 
  59.  59 <p>CodeIgniter uses a modified version of the Active Record Database Pattern.
  60.  60 This pattern allows information to be retrieved, inserted, and updated in your database with minimal scripting.
  61.  61 In some cases only one or two lines of code are necessary to perform a database action.
  62.  62 CodeIgniter does not require that each database table be its own class file.  It instead provides a more simplified interface.</p>
  63.  63 
  64.  64 <p>Beyond simplicity, a major benefit to using the Active Record features is that it allows you to create database independent applications, since the query syntax
  65.  65 is generated by each database adapter.  It also allows for safer queries, since the values are escaped automatically by the system.</p>
  66.  66 
  67.  67 <p class="important"><strong>Note:</strong>  If you intend to write your own queries you can disable this class in your database config file, allowing the core database library and adapter to utilize fewer resources.<br /></p>

  78.  78 <h1><a name="select">&nbsp;</a>Selecting Data</h1>
  79.  79 
  80.  80 <p>The following functions allow you to build SQL <strong>SELECT</strong> statements.</p>
  81.  81 
  82.  82 <p><strong>Note: If you are using PHP 5 you can use method chaining for more compact syntax. This is described at the end of the page.</strong></p>
  83.  83 
  84.  84 
  85.  85 <h2>$this->db->get();</h2>
  86.  86 
  87.  87 <p>Runs the selection query and returns the result.  Can be used by itself to retrieve all records from a table:</p>
  88.  88 
  89.  89 <code>$query = $this->db->get('mytable');<br />
  90.  90 <br />
  91.  91 // Produces: SELECT * FROM mytable</code>
  92.  92 
  93.  93 <p>The second and third parameters enable you to set a limit and offset clause:</p>
  94.  94 
  95.  95 <code>$query = $this->db->get('mytable', 10, 20);<br />
  96.  96 <br />
  97.  97 // Produces: SELECT * FROM mytable LIMIT 20, 10 (in MySQL. Other databases have slightly different syntax)</code>
  98.  98 
  99.  99 <p>You'll notice that the above function is assigned to a variable named <kbd>$query</kbd>, which can be used to show the results:</p>
  100. 100 
  101. 101 <code>$query = $this->db->get('mytable');<br />
  102. 102 <br />
  103. 103 foreach ($query->result() as $row)<br />
  104. 104 {<br />
  105. 105 &nbsp;&nbsp;&nbsp;&nbsp;echo $row->title;<br />
  106. 106 }</code>
  107. 107 
  108. 108 <p>Please visit the <a href="results.html">result functions</a> page for a full discussion regarding result generation.</p>
  109. 109 
  110. 110 
  111. 111 <h2>$this->db->get_where();</h2>
  112. 112 
  113. 113 <p>Identical to the above function except that it permits you to add a "where" clause in the second parameter,
  114. 114 instead of using the db->where() function:</p>
  115. 115 
  116. 116 <code>$query = $this->db->get_where('mytable', array('id' => $id), $limit, $offset);</code>
  117. 117 
  118. 118 <p>Please read the about the where function below for more information.</p>
  119. 119 <p class="important">Note: get_where() was formerly known as getwhere(), which has been removed</p>
  120. 120 
  121. 121 <h2>$this->db->select();</h2>
  122. 122 <p>Permits you to write the SELECT portion of your query:</p>
  123. 123 <p><code>
  124. 124 $this->db->select('title, content, date');<br />
  125. 125 <br />
  126. 126 $query = $this->db->get('mytable');<br />
  127. 127 <br />
  128. 128 // Produces: SELECT title, content, date FROM mytable</code></p>
  129. 129 <p class="important"><strong>Note:</strong> If you are selecting all (*) from a table you do not need to use this function.  When omitted, CodeIgniter assumes you wish to SELECT *</p>
  130. 130 
  131. 131 <p>$this-&gt;db-&gt;select() accepts an optional second parameter. If you set it to FALSE, CodeIgniter will not try to protect your field or table names with backticks. This is useful if you need a compound select statement.</p>
  132. 132 <p><code>$this-&gt;db-&gt;select('(SELECT SUM(payments.amount) FROM payments WHERE payments.invoice_id=4') AS amount_paid', FALSE); <br />
  133. 133 $query = $this-&gt;db-&gt;get('mytable');<br />
  134. 134 </code></p>
  135. 135 <h2>$this->db->select_max();</h2>
  136. 136 <p>Writes a "SELECT MAX(field)" portion for your query. You can optionally include a second parameter to rename the resulting field.</p>
  137. 137 <p><code>
  138. 138 $this->db->select_max('age');<br />
  139. 139 $query = $this->db->get('members');<br />
  140. 140 
  141. 141 // Produces: SELECT MAX(age) as age FROM members<br />
  142. 142 <br />
  143. 143 $this-&gt;db-&gt;select_max('age', 'member_age');<br />
  144. 144 $query = $this-&gt;db-&gt;get('members');<br />
  145. 145 // Produces: SELECT MAX(age) as member_age FROM members</code></p>
  146. 146 
  147. 147 <h2>$this->db->select_min();</h2>
  148. 148 <p>Writes a "SELECT MIN(field)" portion for your query. As with <dfn>select_max()</dfn>, You can optionally include a second parameter to rename the resulting field.</p>
  149. 149 <p><code>
  150. 150 $this->db->select_min('age');<br />
  151. 151 $query = $this->db->get('members');<br />
  152. 152 // Produces: SELECT MIN(age) as age FROM members</code></p>
  153. 153 
  154. 154 <h2>$this->db->select_avg();</h2>
  155. 155 <p>Writes a "SELECT AVG(field)" portion for your query. As with <dfn>select_max()</dfn>, You can optionally include a second parameter to rename the resulting field.</p>
  156. 156 <p><code>
  157. 157 $this->db->select_avg('age');<br />
  158. 158 $query = $this->db->get('members');<br />
  159. 159 // Produces: SELECT AVG(age) as age FROM members</code></p>
  160. 160 
  161. 161 <h2>$this->db->select_sum();</h2>
  162. 162 <p>Writes a "SELECT SUM(field)" portion for your query. As with <dfn>select_max()</dfn>, You can optionally include a second parameter to rename the resulting field.</p>
  163. 163 <p><code>
  164. 164 $this->db->select_sum('age');<br />
  165. 165 $query = $this->db->get('members');<br />
  166. 166 // Produces: SELECT SUM(age) as age FROM members</code></p>
  167. 167 
  168. 168 <h2>$this->db->from();</h2>
  169. 169 
  170. 170 <p>Permits you to write the FROM portion of your query:</p>
  171. 171 
  172. 172 <code>
  173. 173 $this->db->select('title, content, date');<br />
  174. 174 $this->db->from('mytable');<br />
  175. 175 <br />
  176. 176 $query = $this->db->get();<br />
  177. 177 <br />
  178. 178 // Produces: SELECT title, content, date FROM mytable</code>
  179. 179 
  180. 180 <p class="important">Note: As shown earlier, the FROM portion of your query can be specified in the <dfn>$this->db->get()</dfn> function, so use whichever method
  181. 181 you prefer.</p>
  182. 182 
  183. 183 <h2>$this->db->join();</h2>
  184. 184 
  185. 185 <p>Permits you to write the JOIN portion of your query:</p>
  186. 186 
  187. 187 <code>
  188. 188 $this->db->select('*');<br />
  189. 189 $this->db->from('blogs');<br />
  190. 190 $this->db->join('comments', 'comments.id = blogs.id');<br />
  191. 191 <br />
  192. 192 $query = $this->db->get();<br />
  193. 193 <br />
  194. 194 // Produces: <br />
  195. 195 // SELECT * FROM blogs<br />
  196. 196 // JOIN comments ON comments.id = blogs.id<br />
  197. 197 </code>
  198. 198 
  199. 199 <p>Multiple function calls can be made if you need several joins in one query.</p>
  200. 200 
  201. 201 <p>If you need a specific type of JOIN you can specify it via the third parameter of the function.
  202. 202 Options are: left, right, outer, inner, left outer, and right outer.</p>
  203. 203 
  204. 204 <code>
  205. 205 $this->db->join('comments', 'comments.id = blogs.id', <strong>'left'</strong>);<br />
  206. 206 <br />
  207. 207 // Produces: LEFT JOIN comments ON comments.id = blogs.id</code>

  213. 213 <h2>$this->db->where();</h2>
  214. 214 <p>This function enables you to set <strong>WHERE</strong> clauses using one of four methods:</p>
  215. 215 
  216. 216 <p class="important"><strong>Note:</strong> All values passed to this function are escaped automatically, producing safer queries.</p>
  217. 217 
  218. 218 <ol>
  219. 219  <li><strong>Simple key/value method:</strong>
  220. 220 
  221. 221  <code>$this->db->where('name', $name);
  222. 222  <br /><br />// Produces: WHERE name = 'Joe' </code>
  223. 223 
  224. 224  <p>Notice that the equal sign is added for you.</p>
  225. 225 
  226. 226  <p>If you use multiple function calls they will be chained together with <var>AND</var> between them:</p>
  227. 227 
  228. 228  <code>$this->db->where('name', $name);<br />
  229. 229  $this->db->where('title', $title);<br />
  230. 230  $this->db->where('status', $status);
  231. 231  <br /><br />// WHERE name = 'Joe' AND title = 'boss' AND status = 'active' </code> </li>
  232. 232 
  233. 233  <li><strong>Custom key/value method:</strong>
  234. 234 
  235. 235  <p>You can include an operator in the first parameter in order to control the comparison:</p>
  236. 236 
  237. 237  <code>$this->db->where('name !=', $name);<br />
  238. 238  $this->db->where('id <', $id);
  239. 239  <br /><br />// Produces: WHERE name != 'Joe' AND id < 45 </code> </li>
  240. 240  <li><strong>Associative array method:</strong>
  241. 241 
  242. 242 
  243. 243  <code>
  244. 244  $array = array('name' => $name, 'title' => $title, 'status' => $status);<br /><br />
  245. 245 
  246. 246  $this->db->where($array);
  247. 247  <br /><br />// Produces: WHERE name = 'Joe' AND title = 'boss' AND status = 'active' </code>
  248. 248 
  249. 249  <p>You can include your own operators using this method as well:</p>
  250. 250 
  251. 251  <code>
  252. 252  $array = array('name !=' => $name, 'id <' => $id, 'date >' => $date);<br /><br />
  253. 253 
  254. 254  $this->db->where($array);</code> </li>
  255. 255  <li><strong>Custom string:</strong>
  256. 256 
  257. 257  <p>You can write your own clauses manually:</p>
  258. 258 
  259. 259  <code>
  260. 260  $where = "name='Joe' AND status='boss' OR status='active'";<br /><br />
  261. 261  $this->db->where($where);</code></li>
  262. 262  </ol>
  263. 263 
  264. 264 
  265. 265 <p>$this-&gt;db-&gt;where() accepts an optional third parameter. If you set it to FALSE, CodeIgniter will not try to protect your field or table names with backticks.</p>
  266. 266 <p><code> $this-&gt;db-&gt;where('MATCH (field) AGAINST (&quot;value&quot;)', NULL, FALSE);<br />
  267. 267 </code></p>
  268. 268 <h2>$this->db->or_where();</h2>
  269. 269 <p>This function is identical to the one above, except that multiple instances are joined by OR:</p>
  270. 270 
  271. 271 <code>
  272. 272 $this->db->where('name !=', $name);<br />
  273. 273 $this->db->or_where('id >', $id);
  274. 274 <br />
  275. 275 <br />// Produces: WHERE name != 'Joe' OR id > 50</code>
  276. 276 
  277. 277 <p class="important">Note: or_where() was formerly known as orwhere(), which has been removed.</p>
  278. 278 
  279. 279 
  280. 280 <h2>$this->db->where_in();</h2>
  281. 281 <p>Generates a WHERE field IN ('item', 'item') SQL query joined with  AND if appropriate</p>
  282. 282 <p><code>
  283. 283  $names = array('Frank', 'Todd', 'James');<br />
  284. 284  $this->db->where_in('username', $names);<br />
  285. 285  // Produces: WHERE username IN ('Frank', 'Todd', 'James')</code></p>
  286. 286 
  287. 287 <h2>$this->db->or_where_in();</h2>
  288. 288 <p>Generates a WHERE field IN ('item', 'item') SQL query joined with OR if appropriate</p>
  289. 289 <p><code>
  290. 290  $names = array('Frank', 'Todd', 'James');<br />
  291. 291  $this->db->or_where_in('username', $names);<br />
  292. 292  // Produces: OR username IN ('Frank', 'Todd', 'James')</code></p>
  293. 293 
  294. 294 <h2>$this->db->where_not_in();</h2>
  295. 295 <p>Generates a WHERE field NOT IN ('item', 'item') SQL query joined with AND if appropriate</p>
  296. 296 <p><code>
  297. 297  $names = array('Frank', 'Todd', 'James');<br />
  298. 298  $this->db->where_not_in('username', $names);<br />
  299. 299  // Produces: WHERE username NOT IN ('Frank', 'Todd', 'James')</code></p>
  300. 300 
  301. 301 <h2>$this->db->or_where_not_in();</h2>
  302. 302 <p>Generates a WHERE field NOT IN ('item', 'item') SQL query joined with OR if appropriate</p>
  303. 303 <p><code>
  304. 304  $names = array('Frank', 'Todd', 'James');<br />
  305. 305  $this->db->or_where_not_in('username', $names);<br />
  306. 306  // Produces: OR username NOT IN ('Frank', 'Todd', 'James')</code></p>
  307. 307 
  308. 308 <h2>$this->db->like();</h2>
  309. 309 <p>This function enables you to generate <strong>LIKE</strong> clauses, useful for doing searches.</p>
  310. 310 
  311. 311 <p class="important"><strong>Note:</strong> All values passed to this function are escaped automatically.</p>
  312. 312 
  313. 313 
  314. 314 <ol>
  315. 315  <li><strong>Simple key/value method:</strong>
  316. 316 
  317. 317  <code>$this->db->like('title', 'match');
  318. 318  <br /><br />// Produces: WHERE title LIKE '%match%' </code>
  319. 319 
  320. 320  <p>If you use multiple function calls they will be chained together with <var>AND</var> between them:</p>
  321. 321 
  322. 322  <code>$this->db->like('title', 'match');<br />
  323. 323  $this->db->like('body', 'match');
  324. 324  <br /><br />
  325. 325  // WHERE title LIKE '%match%' AND  body LIKE '%match%</code>
  326. 326  If you want to control where the wildcard (%) is placed, you can use an optional third argument. Your options are 'before', 'after' and 'both' (which is the default).
  327. 327  <code>$this->db->like('title', 'match', 'before');
  328. 328  <br />
  329. 329  // Produces: WHERE title LIKE '%match' <br />
  330. 330  <br />
  331. 331  $this-&gt;db-&gt;like('title', 'match', 'after'); <br />

  678. 678 &nbsp;&nbsp;&nbsp;array(<br />
  679. 679 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'title' => 'My title' ,<br />
  680. 680 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'name' => 'My Name 2' ,<br />
  681. 681 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'date' => 'My date 2'<br />
  682. 682 &nbsp;&nbsp;&nbsp;),<br />
  683. 683 &nbsp;&nbsp;&nbsp;array(<br />
  684. 684 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'title' => 'Another title' ,<br />
  685. 685 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'name' => 'Another Name 2' ,<br />
  686. 686 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'date' => 'Another date 2'<br />
  687. 687 &nbsp;&nbsp;&nbsp;)<br/>
  688. 688 );<br />
  689. 689 <br />
  690. 690 $this->db->update_batch('mytable', $data, 'title');
  691. 691 <br /><br />
  692. 692 // Produces: <br />
  693. 693 // UPDATE `mytable` SET `name` = CASE<br />
  694. 694 // WHEN `title` = 'My title' THEN 'My Name 2'<br />
  695. 695 // WHEN `title` = 'Another title' THEN 'Another Name 2'<br />
  696. 696 // ELSE `name` END,<br />
  697. 697 // `date` = CASE <br />
  698. 698 // WHEN `title` = 'My title' THEN 'My date 2'<br />
  699. 699 // WHEN `title` = 'Another title' THEN 'Another date 2'<br />
  700. 700 // ELSE `date` END<br />
  701. 701 // WHERE `title` IN ('My title','Another title')</code>
  702. 702 
  703. 703 <p>The first parameter will contain the table name, the second is an associative array of values, the third parameter is the where key.</p>
  704. 704 
  705. 705 <p class="important"><strong>Note:</strong> All values are escaped automatically producing safer queries.</p>
  706. 706 
  707. 707 
  708. 708 <a name="delete">&nbsp;</a>
  709. 709 <h1>Deleting Data</h1>
  710. 710 
  711. 711 
  712. 712 
  713. 713 <h2>$this->db->delete();</h2>
  714. 714 <p>Generates a delete SQL string and runs the query.</p>
  715. 715 
  716. 716 <code>
  717. 717 $this->db->delete('mytable', array('id' => $id));
  718. 718 <br /><br />
  719. 719 // Produces:<br />
  720. 720 // DELETE FROM mytable <br />
  721. 721 // WHERE id = $id</code>
  722. 722 
  723. 723 <p>The first parameter is the table name, the second is the where clause. You can also use the <dfn>where()</dfn> or <dfn>or_where()</dfn> functions instead of passing
  724. 724 the data to the second parameter of the function:</p>
  725. 725 
  726. 726 <p><code> $this->db->where('id', $id);<br />
  727. 727  $this->db->delete('mytable'); <br />
  728. 728  <br />
  729. 729  // Produces:<br />
  730. 730  // DELETE FROM mytable <br />
  731. 731  // WHERE id = $id</code></p>
  732. 732 <p>An array of table names can be passed into delete() if you would like to delete data from more than 1 table.</p>
  733. 733 <p><code>$tables = array('table1', 'table2', 'table3');<br />
  734. 734 $this-&gt;db-&gt;where('id', '5');<br />
  735. 735 $this-&gt;db-&gt;delete($tables);</code></p>
  736. 736 <p>If you want to delete all data from a table, you can use the <dfn>truncate()</dfn> function, or <dfn>empty_table()</dfn>.</p>
  737. 737 <h2>$this-&gt;db-&gt;empty_table();</h2>
  738. 738 <p>Generates a delete SQL string and runs the query.<code> $this-&gt;db-&gt;empty_table('mytable'); <br />
  739. 739  <br />
  740. 740 // Produces<br />
  741. 741 // DELETE FROM mytable</code></p>
  742. 742 <h2>$this-&gt;db-&gt;truncate();</h2>
  743. 743 <p>Generates a truncate SQL string and runs the query.</p>
  744. 744 <code> $this-&gt;db-&gt;from('mytable'); <br />
  745. 745 $this-&gt;db-&gt;truncate(); <br />
  746. 746 // or <br />
  747. 747 $this-&gt;db-&gt;truncate('mytable'); <br />
  748. 748 <br />
  749. 749 // Produce:<br />
  750. 750 // TRUNCATE mytable <br />
  751. 751 </code>
  752. 752 <p class="important"><strong>Note:</strong> If the TRUNCATE command isn't available, truncate() will execute as &quot;DELETE FROM table&quot;.</p>
  753. 753 
  754. 754 <h1><a name="chaining">&nbsp;</a>Method Chaining</h1>
  755. 755 
  756. 756 <p>Method chaining allows you to simplify your syntax by connecting multiple functions.  Consider this example:</p>
  757. 757 
  758. 758 <code>
  759. 759 <dfn>$this->db</dfn><kbd>-></kbd><var>select</var>('title')<kbd>-></kbd><var>from</var>('mytable')<kbd>-></kbd><var>where</var>('id', $id)<kbd>-></kbd><var>limit</var>(10, 20);<br />
  760. 760 <br />
  761. 761 $query = $this->db->get();</code>
  762. 762 
  763. 763 <p class="important"><strong>Note:</strong> Method chaining only works with PHP 5.</p>
  764. 764 
  765. 765 <p>&nbsp;</p>
  766. 766 
  767. 767 <h1><a name="caching">&nbsp;</a>Active Record Caching</h1>
  768. 768 
  769. 769 <p>While not &quot;true&quot; caching, Active Record enables you to save (or &quot;cache&quot;) certain parts of your queries for reuse at a later point in your script's execution. Normally, when an Active Record call is completed, all stored information is reset for the next call. With caching, you can prevent this reset, and reuse information easily.</p>
  770. 770 
  771. 771 <p>Cached calls are cumulative. If you make 2 cached select() calls, and then 2 uncached select() calls, this will result in 4 select() calls. There are three Caching functions available:</p>
  772. 772 
  773. 773 <h2>$this-&gt;db-&gt;start_cache()</h2>
  774. 774 
  775. 775 <p>This function must be called to begin caching. All Active Record queries of the correct type (see below for supported queries) are stored for later use.</p>
  776. 776 
  777. 777 <h2>$this-&gt;db-&gt;stop_cache()</h2>
  778. 778 
  779. 779 <p>This function can be called to stop caching.</p>
  780. 780 
  781. 781 <h2>$this-&gt;db-&gt;flush_cache()</h2>
  782. 782 
  783. 783 <p>This function deletes all items from the Active Record cache.</p>
  784. 784 
  785. 785 <p>Here's a usage example:</p>
  786. 786 
  787. 787 <p><code>$this-&gt;db-&gt;start_cache();<br />
  788. 788 $this-&gt;db-&gt;select('field1');<br />
  789. 789 $this-&gt;db-&gt;stop_cache();<br /><br />
  790. 790 $this-&gt;db-&gt;get('tablename');<br />
  791. 791 <br />
  792. 792 //Generates: SELECT `field1` FROM (`tablename`)<br />
  793. 793 <br />
  794. 794 $this-&gt;db-&gt;select('field2');<br />
  795. 795 $this-&gt;db-&gt;get('tablename');<br />
  796. 796 <br />
  797. 797 //Generates:  SELECT `field1`, `field2` FROM (`tablename`)<br />
  798. 798 <br />
  799. 799 $this-&gt;db-&gt;flush_cache();<br />
  800. 800 <br />
  801. 801 $this-&gt;db-&gt;select('field2');<br />
  802. 802 $this-&gt;db-&gt;get('tablename');<br />
  803. 803 <br />
  804. 804 //Generates:  SELECT `field2` FROM (`tablename`)</code></p>
  805. 805 
  806. 806 <p class="important"> <strong>Note:</strong> The following statements can be cached: select, from, join, where, like, group_by, having, order_by, set</p>
