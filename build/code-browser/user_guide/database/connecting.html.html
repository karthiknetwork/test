  61.  61 <h1>Connecting to your Database</h1>
  62.  62 
  63.  63 <p>There are two ways to connect to a database:</p>
  64.  64 
  65.  65 <h2>Automatically Connecting</h2>
  66.  66 
  67.  67 <p>The "auto connect" feature will load and instantiate the database class with every page load.
  68.  68 To enable "auto connecting", add the word <var>database</var> to the library array, as indicated in the following file:</p>
  69.  69 
  70.  70 <p><kbd>application/config/autoload.php</kbd></p>
  71.  71 
  72.  72 <h2>Manually Connecting</h2>
  73.  73 
  74.  74 <p>If only some of your pages require database connectivity you can manually connect to your database by adding this
  75.  75 line of code in any function where it is needed, or in your class constructor to make the database
  76.  76 available globally in that class.</p>
  77.  77 
  78.  78 <code>$this->load->database();</code>
  79.  79 
  80.  80 <p class="important">If the above function does <strong>not</strong> contain any information in the first parameter it will connect
  81.  81 to the group specified in your database config file. For most people, this is the preferred method of use.</p>
  82.  82 
  83.  83 <h3>Available Parameters</h3>
  84.  84 
  85.  85 <ol>
  86.  86  <li>The database connection values, passed either as an array or a DSN string.</li>
  87.  87  <li>TRUE/FALSE (boolean).  Whether to return the connection ID (see Connecting to Multiple Databases below).</li>
  88.  88  <li>TRUE/FALSE (boolean).  Whether to enable the Active Record class.  Set to TRUE by default.</li>
  89.  89 </ol>
  90.  90 
  91.  91 
  92.  92 <h3>Manually Connecting to a Database</h3>
  93.  93 
  94.  94 <p>The first parameter of this function can <strong>optionally</strong> be used to specify a particular database group
  95.  95 from your config file, or you can even submit connection values for a database that is not specified in your config file.
  96.  96 Examples:</p>
  97.  97 
  98.  98 <p>To choose a specific group from your config file you can do this:</p>
  99.  99 
  100. 100 <code>$this->load->database('<samp>group_name</samp>');</code>
  101. 101 
  102. 102 <p>Where <samp>group_name</samp> is the name of the connection group from your config file.</p>
  103. 103 
  104. 104 
  105. 105 <p>To connect manually to a desired database you can pass an array of values:</p>
  106. 106 
  107. 107 <code>$config['hostname'] = "localhost";<br />
  108. 108 $config['username'] = "myusername";<br />
  109. 109 $config['password'] = "mypassword";<br />
  110. 110 $config['database'] = "mydatabase";<br />
  111. 111 $config['dbdriver'] = "mysql";<br />
  112. 112 $config['dbprefix'] = "";<br />
  113. 113 $config['pconnect'] = FALSE;<br />
  114. 114 $config['db_debug'] = TRUE;<br />
  115. 115 $config['cache_on'] = FALSE;<br />
  116. 116 $config['cachedir'] = "";<br />
  117. 117 $config['char_set'] = "utf8";<br />
  118. 118 $config['dbcollat'] = "utf8_general_ci";<br />
  119. 119 <br />
  120. 120 $this->load->database(<samp>$config</samp>);</code>
  121. 121 
  122. 122 <p>For information on each of these values please see the <a href="configuration.html">configuration page</a>.</p>
  123. 123 
  124. 124 <p>Or you can submit your database values as a Data Source Name. DSNs must have this prototype:</p>
  125. 125 
  126. 126 <code>$dsn = 'dbdriver://username:password@hostname/database';<br />
  127. 127 <br />
  128. 128 $this->load->database(<samp>$dsn</samp>);</code>
  129. 129 
  130. 130 <p>To override default config values when connecting with a DSN string, add the config variables as a query string.</p>
  131. 131 
  132. 132 <code>$dsn = 'dbdriver://username:password@hostname/database?char_set=utf8&amp;dbcollat=utf8_general_ci&amp;cache_on=true&amp;cachedir=/path/to/cache';<br />
  133. 133 <br />
  134. 134 $this->load->database(<samp>$dsn</samp>);</code>
  135. 135 
  136. 136 <h2>Connecting to Multiple Databases</h2>
  137. 137 
  138. 138 <p>If you need to connect to more than one database simultaneously you can do so as follows:</p>
  139. 139 
  140. 140 
  141. 141 <code>$DB1 = $this->load->database('group_one', TRUE);<br />
  142. 142 $DB2 = $this->load->database('group_two', TRUE);
  143. 143 </code>
  144. 144 
  145. 145 <p>Note: Change the words "group_one" and "group_two" to the specific group names you are connecting to (or
  146. 146 you can pass the connection values as indicated above).</p>
  147. 147 
  148. 148 <p>By setting the second parameter to TRUE (boolean) the function will return the database object.</p>
  149. 149 
  150. 150 <div class="important">
  151. 151 <p>When you connect this way, you will use your object name to issue commands rather than the syntax used throughout this guide.  In other words, rather than issuing commands with:</p>
  152. 152 
  153. 153 <p>$this->db->query();<br />$this->db->result();<br /> etc...</p>
  154. 154 
  155. 155 <p>You will instead use:</p>
  156. 156 
  157. 157 <p>$DB1->query();<br />$DB1->result();<br /> etc...</p>
  158. 158 
  159. 159 </div>
  160. 160 
  161. 161 <h2>Reconnecting / Keeping the Connection Alive</h2>
  162. 162 
  163. 163 <p>If the database server's idle timeout is exceeded while you're doing some heavy PHP lifting (processing an image, for instance), you should consider pinging the server by using the <dfn>reconnect()</dfn> method before sending further queries, which can gracefully keep the connection alive or re-establish it.</p>
  164. 164 
  165. 165 <code>$this->db->reconnect();</code>
  166. 166 
  167. 167 <h2>Manually closing the Connection</h2>
  168. 168 
  169. 169 <p>While CodeIgniter intelligently takes care of closing your database connections, you can explicitly close the connection.</p>
  170. 170 
  171. 171 <code>$this->db->close();</code>
