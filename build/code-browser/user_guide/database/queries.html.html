  62.  62 <h1>Queries</h1>
  63.  63 
  64.  64 <h2>$this->db->query();</h2>
  65.  65 
  66.  66 <p>To submit a query, use the following function:</p>
  67.  67 
  68.  68 <code>$this->db->query('YOUR QUERY HERE');</code>
  69.  69 
  70.  70 <p>The <dfn>query()</dfn> function returns a database result <strong>object</strong> when "read" type queries are run,
  71.  71 which you can use to <a href="results.html">show your results</a>.  When "write" type queries are run it simply returns TRUE or FALSE
  72.  72 depending on success or failure.  When retrieving data you will typically assign the query to your own variable, like this:</p>
  73.  73 
  74.  74 <code><var>$query</var> = $this->db->query('YOUR QUERY HERE');</code>
  75.  75 
  76.  76 <h2>$this->db->simple_query();</h2>
  77.  77 
  78.  78 <p>This is a simplified version of the <dfn>$this->db->query()</dfn> function.  It ONLY returns TRUE/FALSE on success or failure.
  79.  79 It DOES NOT return a database result set, nor does it set the query timer, or compile bind data, or store your query for debugging.
  80.  80 It simply lets you submit a query. Most users will rarely use this function.</p>
  81.  81

  83.  83 <h1>Working with Database prefixes manually</h1>
  84.  84 <p>If you have configured a database prefix and would like to prepend it to a table name for use in a native SQL query for example, then you can use the following:</p>
  85.  85 <p><code>$this-&gt;db-&gt;dbprefix('tablename');<br />
  86.  86 // outputs prefix_tablename</code></p>
  87.  87 
  88.  88 <p>If for any reason you would like to change the prefix programatically without needing to create a new connection, you can use this method:</p>
  89.  89 <p><code>$this-&gt;db-&gt;set_dbprefix('newprefix');<br /><br />
  90.  90 $this-&gt;db-&gt;dbprefix('tablename');<br />
  91.  91 // outputs newprefix_tablename</code></p>
  92.  92

  94.  94 <h1>Protecting identifiers</h1>
  95.  95 <p>In many databases it is advisable to protect table and field names - for example with backticks in MySQL. <strong>Active Record queries are automatically protected</strong>, however if you need to manually protect an identifier you can use:</p>
  96.  96 <p><code>$this-&gt;db-&gt;protect_identifiers('table_name');</code></p>
  97.  97 
  98.  98 <p>This function will also add a table prefix to your table, assuming you have a prefix specified in your database config file. To enable the prefixing set <kbd>TRUE</kbd> (boolen) via the second parameter:</p>
  99.  99 <p><code>$this-&gt;db-&gt;protect_identifiers('table_name', <kbd>TRUE</kbd>);</code></p>

  101. 101 
  102. 102 <h1>Escaping Queries</h1>
  103. 103 <p>It's a very good security practice to escape your data before submitting it into your database.
  104. 104 CodeIgniter has three methods that help you do this:</p>
  105. 105 
  106. 106 <ol>
  107. 107 <li><strong>$this->db->escape()</strong> This function determines the data type so that it
  108. 108 can escape only string data.  It also automatically adds single quotes around the data so you don't have to:
  109. 109 
  110. 110 <code>$sql = "INSERT INTO table (title) VALUES(".$this->db->escape($title).")";</code></li>
  111. 111 
  112. 112 <li><strong>$this->db->escape_str()</strong>  This function escapes the data passed to it, regardless of type.
  113. 113 Most of the time you'll use the above function rather than this one. Use the function like this:
  114. 114 
  115. 115 <code>$sql = "INSERT INTO table (title) VALUES('".$this->db->escape_str($title)."')";</code></li>
  116. 116 
  117. 117 <li><strong>$this->db->escape_like_str()</strong>  This method should be used when strings are to be used in LIKE
  118. 118 conditions so that LIKE wildcards ('%', '_') in the string are also properly escaped.
  119. 119 
  120. 120 <code>$search = '20% raise';<br />
  121. 121 $sql = "SELECT id FROM table WHERE column LIKE '%".$this->db->escape_like_str($search)."%'";</code></li>
  122. 122 
  123. 123 </ol>

  126. 126 <h1>Query Bindings</h1>
  127. 127 
  128. 128 
  129. 129 <p>Bindings enable you to simplify your query syntax by letting the system put the queries together for you. Consider the following example:</p>
  130. 130 
  131. 131 <code>
  132. 132 $sql = "SELECT * FROM some_table WHERE id = <var>?</var> AND status = <var>?</var> AND author = <var>?</var>";
  133. 133 <br /><br />
  134. 134 $this->db->query($sql, array(3, 'live', 'Rick'));
  135. 135 </code>
  136. 136 
  137. 137 <p>The question marks in the query are automatically replaced with the values in the array in the second parameter of the query function.</p>
  138. 138 <p class="important">The secondary benefit of using binds is that the values are automatically escaped, producing safer queries.  You don't have to remember to manually escape data; the engine does it automatically for you.</p>
