  59.  59 <h1>Web Page Caching</h1>
  60.  60 
  61.  61 <p>CodeIgniter lets you cache your pages in order to achieve maximum performance.</p>
  62.  62 
  63.  63 <p>Although CodeIgniter is quite fast, the amount of dynamic information you display in your pages will correlate directly to the
  64.  64 server resources, memory, and processing cycles utilized, which affect your page load speeds.
  65.  65 By caching your pages, since they are saved in their fully rendered state, you can achieve performance that nears that of static web pages.</p>
  66.  66 
  67.  67 
  68.  68 <h2>How Does Caching Work?</h2>
  69.  69 
  70.  70 <p>Caching can be enabled on a per-page basis, and you can set the length of time that a page should remain cached before being refreshed.
  71.  71 When a page is loaded for the first time, the cache file will be written to your <dfn>application/cache</dfn> folder.  On subsequent page loads the cache file will be retrieved
  72.  72 and sent to the requesting user's browser.  If it has expired, it will be deleted and refreshed before being sent to the browser.</p>
  73.  73 
  74.  74 <p>Note: The Benchmark tag is not cached so you can still view your page load speed when caching is enabled.</p>
  75.  75 
  76.  76 <h2>Enabling Caching</h2>
  77.  77 
  78.  78 <p>To enable caching, put the following tag in any of your controller functions:</p>
  79.  79 
  80.  80 <code>$this->output->cache(<var>n</var>);</code>
  81.  81 
  82.  82 <p>Where <var>n</var> is the number of <strong>minutes</strong> you wish the page to remain cached between refreshes.</p>
  83.  83 
  84.  84 <p>The above tag can go anywhere within a function. It is not affected by the order that it appears, so place it wherever it seems
  85.  85 most logical to you. Once the tag is in place, your pages will begin being cached.</p>
  86.  86 
  87.  87 <p class="important"><strong>Warning:</strong> Because of the way CodeIgniter stores content for output, caching will only work if you are generating display for your controller with a <a href="./views.html">view</a>.</p>
  88.  88 <p class="important"><strong>Note:</strong> Before the cache files can be written you must set the file permissions on your
  89.  89 <dfn>application/cache</dfn> folder such that it is writable.</p>
  90.  90 
  91.  91 <h2>Deleting Caches</h2>
  92.  92 
  93.  93 <p>If you no longer wish to cache a file you can remove the caching tag and it will no longer be refreshed when it expires.  Note:
  94.  94 Removing the tag will not delete the cache immediately.  It will have to expire normally.  If you need to remove it earlier you
  95.  95 will need to manually delete it from your cache folder.</p>
