  79.  79 <h2>What is a Controller?</h2>
  80.  80 
  81.  81 <p><dfn>A Controller is simply a class file that is named in a way that can be associated with a URI.</dfn></p>
  82.  82 
  83.  83 <p>Consider this URI:</p>
  84.  84 
  85.  85 <code>example.com/index.php/<var>blog</var>/</code>
  86.  86 
  87.  87 <p>In the above example, CodeIgniter would attempt to find a controller named <dfn>blog.php</dfn> and load it.</p>
  88.  88 
  89.  89 <p><strong>When a controller's name matches the first segment of a URI, it will be loaded.</strong></p>
  90.  90 
  91.  91 <a name="hello"></a>
  92.  92 <h2>Let's try it:&nbsp; Hello World!</h2>
  93.  93 
  94.  94 <p>Let's create a simple controller so you can see it in action.  Using your text editor, create a file called <dfn>blog.php</dfn>, and put the following code in it:</p>
  95.  95 
  96.  96 
  97.  97 <textarea class="textarea" style="width:100%" cols="50" rows="10">
  98.  98 &lt;?php
  99.  99 class Blog extends CI_Controller {
  100. 100 
  101. 101  public function index()
  102. 102  {
  103. 103  echo 'Hello World!';
  104. 104  }
  105. 105 }
  106. 106 ?&gt;
  107. 107 </textarea>
  108. 108 
  109. 109 
  110. 110 
  111. 111 <p>Then save the file to your <dfn>application/controllers/</dfn> folder.</p>
  112. 112 
  113. 113 <p>Now visit the your site using a URL similar to this:</p>
  114. 114 
  115. 115 <code>example.com/index.php/<var>blog</var>/</code>
  116. 116 
  117. 117 <p>If you did it right, you should see <samp>Hello World!</samp>.</p>
  118. 118 
  119. 119 <p>Note: Class names must start with an uppercase letter.  In other words, this is valid:</p>
  120. 120 
  121. 121 <code>&lt;?php<br />
  122. 122 class <var>Blog</var> extends CI_Controller {<br />
  123. 123 <br />
  124. 124 }<br />
  125. 125 ?&gt;</code>
  126. 126 
  127. 127 <p>This is <strong>not</strong> valid:</p>
  128. 128 
  129. 129 <code>&lt;?php<br />
  130. 130 class <var>blog</var> extends CI_Controller {<br />
  131. 131 <br />
  132. 132 }<br />
  133. 133 ?&gt;</code>
  134. 134 
  135. 135 <p>Also, always make sure your controller <dfn>extends</dfn> the parent controller class so that it can inherit all its functions.</p>
  136. 136 
  137. 137 
  138. 138 
  139. 139 <a name="functions"></a>
  140. 140 <h2>Functions</h2>
  141. 141 
  142. 142 <p>In the above example the function name is <dfn>index()</dfn>.  The "index" function is always loaded by default if the
  143. 143 <strong>second segment</strong> of the URI is empty.  Another way to show your "Hello World" message would be this:</p>
  144. 144 
  145. 145 <code>example.com/index.php/<var>blog</var>/<samp>index</samp>/</code>
  146. 146 
  147. 147 <p><strong>The second segment of the URI determines which function in the controller gets called.</strong></p>
  148. 148 
  149. 149 <p>Let's try it.  Add a new function to your controller:</p>
  150. 150 
  151. 151 
  152. 152 <textarea class="textarea" style="width:100%" cols="50" rows="15">
  153. 153 &lt;?php
  154. 154 class Blog extends CI_Controller {
  155. 155 
  156. 156  public function index()
  157. 157  {
  158. 158  echo 'Hello World!';
  159. 159  }
  160. 160 
  161. 161  public function comments()
  162. 162  {
  163. 163  echo 'Look at this!';
  164. 164  }
  165. 165 }
  166. 166 ?&gt;
  167. 167 </textarea>
  168. 168 
  169. 169 <p>Now load the following URL to see the <dfn>comment</dfn> function:</p>
  170. 170 
  171. 171 <code>example.com/index.php/<var>blog</var>/<samp>comments</samp>/</code>
  172. 172 
  173. 173 <p>You should see your new message.</p>
  174. 174 
  175. 175 <a name="passinguri"></a>
  176. 176 <h2>Passing URI Segments to your Functions</h2>
  177. 177 
  178. 178 <p>If your URI contains more then two segments they will be passed to your function as parameters.</p>
  179. 179 
  180. 180 <p>For example, lets say you have a URI like this:</p>
  181. 181 
  182. 182 <code>example.com/index.php/<var>products</var>/<samp>shoes</samp>/<kbd>sandals</kbd>/<dfn>123</dfn></code>
  183. 183 
  184. 184 <p>Your function will be passed URI segments 3 and 4 ("sandals" and "123"):</p>
  185. 185 
  186. 186 <code>
  187. 187 &lt;?php<br />
  188. 188 class Products extends CI_Controller {<br />
  189. 189 <br />
  190. 190 &nbsp;&nbsp;&nbsp;&nbsp;public function shoes($sandals, $id)<br />
  191. 191 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  192. 192 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;echo $sandals;<br />
  193. 193 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;echo $id;<br />
  194. 194 &nbsp;&nbsp;&nbsp;&nbsp;}<br />
  195. 195 }<br />
  196. 196 ?&gt;
  197. 197 </code>
  198. 198 
  199. 199 <p class="important"><strong>Important:</strong>&nbsp; If you are using the <a href="routing.html">URI Routing</a> feature, the segments
  200. 200 passed to your function will be the re-routed ones.</p>
  201. 201 
  202. 202 
  203. 203 <a name="default"></a>
  204. 204 <h2>Defining a Default Controller</h2>
  205. 205 
  206. 206 <p>CodeIgniter can be told to load a default controller when a URI is not present,
  207. 207 as will be the case when only your site root URL is requested.  To specify a default controller, open
  208. 208 your <dfn>application/config/routes.php</dfn> file and set this variable:</p>
  209. 209 
  210. 210 <code>$route['default_controller'] = '<var>Blog</var>';</code>
  211. 211 
  212. 212 <p>Where <var>Blog</var> is the name of the controller class you want used. If you now load your main index.php file without
  213. 213 specifying any URI segments you'll see your Hello World message by default.</p>
  214. 214 
  215. 215 
  216. 216 
  217. 217 <a name="remapping"></a>
  218. 218 <h2>Remapping Function Calls</h2>
  219. 219 
  220. 220 <p>As noted above, the second segment of the URI typically determines which function in the controller gets called.
  221. 221 CodeIgniter permits you to override this behavior through the use of the <kbd>_remap()</kbd> function:</p>
  222. 222 
  223. 223 <code>public function _remap()<br />
  224. 224 {<br />
  225. 225 &nbsp;&nbsp;&nbsp;&nbsp;// Some code here...<br />
  226. 226 }</code>
  227. 227 
  228. 228 <p class="important"><strong>Important:</strong>&nbsp; If your controller contains a function named <kbd>_remap()</kbd>, it will <strong>always</strong>
  229. 229 get called regardless of what your URI contains.  It overrides the normal behavior in which the URI determines which function is called,
  230. 230 allowing you to define your own function routing rules.</p>
  231. 231 
  232. 232 <p>The overridden function call (typically the second segment of the URI) will be passed as a parameter to the <kbd>_remap()</kbd> function:</p>
  233. 233 
  234. 234 <code>public function _remap(<var>$method</var>)<br />
  235. 235 {<br />
  236. 236 &nbsp;&nbsp;&nbsp;&nbsp;if ($method == 'some_method')<br />
  237. 237 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  238. 238 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$this->$method();<br />
  239. 239 &nbsp;&nbsp;&nbsp;&nbsp;}<br />
  240. 240 &nbsp;&nbsp;&nbsp;&nbsp;else<br />
  241. 241 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  242. 242 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$this->default_method();<br />
  243. 243 &nbsp;&nbsp;&nbsp;&nbsp;}<br />
  244. 244 }</code>
  245. 245 
  246. 246 <p>Any extra segments after the method name are passed into <kbd>_remap()</kbd> as an optional second parameter. This array can be used in combination with PHP's <a href="http://php.net/call_user_func_array">call_user_func_array</a> to emulate CodeIgniter's default behavior.</p>
  247. 247 
  248. 248 <code>public function _remap($method, $params = array())<br />
  249. 249 {<br />
  250. 250 &nbsp;&nbsp;&nbsp;&nbsp;$method = 'process_'.$method;<br />
  251. 251 &nbsp;&nbsp;&nbsp;&nbsp;if (method_exists($this, $method))<br />
  252. 252 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  253. 253 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return call_user_func_array(array($this, $method), $params);<br />
  254. 254 &nbsp;&nbsp;&nbsp;&nbsp;}<br />
  255. 255 &nbsp;&nbsp;&nbsp;&nbsp;show_404();<br />
  256. 256 }</code>
  257. 257 
  258. 258 
  259. 259 <a name="output"></a>
  260. 260 <h2>Processing Output</h2>
  261. 261 
  262. 262 <p>CodeIgniter has an output class that takes care of sending your final rendered data to the web browser automatically.  More information on this can be found in the
  263. 263 <a href="views.html">Views</a> and <a href="../libraries/output.html">Output class</a> pages.  In some cases, however, you might want to
  264. 264 post-process the finalized data in some way and send it to the browser yourself.  CodeIgniter permits you to
  265. 265 add a function named <dfn>_output()</dfn> to your controller that will receive the finalized output data.</p>
  266. 266 
  267. 267 <p><strong>Important:</strong>&nbsp; If your controller contains a function named <kbd>_output()</kbd>, it will <strong>always</strong>
  268. 268 be called by the output class instead of echoing the finalized data directly. The first parameter of the function will contain the finalized output.</p>
  269. 269 
  270. 270 <p>Here is an example:</p>
  271. 271 
  272. 272 <code>
  273. 273 public function _output($output)<br />
  274. 274 {<br />
  275. 275 &nbsp;&nbsp;&nbsp;&nbsp;echo $output;<br />
  276. 276 }</code>
  277. 277 
  278. 278 <p class="important">Please note that your <dfn>_output()</dfn> function will receive the data in its finalized state.  Benchmark and memory usage data will be rendered,
  279. 279 cache files written (if you have caching enabled), and headers will be sent (if you use that <a href="../libraries/output.html">feature</a>)
  280. 280 before it is handed off to the _output() function.<br />
  281. 281 <br />
  282. 282 To have your controller's output cached properly, its <dfn>_output()</dfn> method can use:<br />
  283. 283 
  284. 284 <code>if ($this-&gt;output-&gt;cache_expiration &gt; 0)<br />
  285. 285 {<br />
  286. 286 &nbsp;&nbsp;&nbsp;&nbsp;$this-&gt;output-&gt;_write_cache($output);<br />
  287. 287 }</code>
  288. 288 
  289. 289 If you are using this feature the page execution timer and memory usage stats might not be perfectly accurate
  290. 290 since they will not take into acccount any further processing you do.  For an alternate way to control output <em>before</em> any of the final processing is done, please see
  291. 291 the available methods in the <a href="../libraries/output.html">Output Class</a>.</p>
  292. 292 
  293. 293 <a name="private"></a>
  294. 294 <h2>Private Functions</h2>
  295. 295 
  296. 296 
  297. 297 <p>In some cases you may want certain functions hidden from public access.  To make a function private, simply add an
  298. 298 underscore as the name prefix and it will not be served via a URL request. For example, if you were to have a function like this:</p>
  299. 299 
  300. 300 <code>
  301. 301 private function _utility()<br />
  302. 302 {<br />
  303. 303 &nbsp;&nbsp;// some code<br />
  304. 304 }</code>
  305. 305 
  306. 306 <p>Trying to access it via the URL, like this, will not work:</p>
  307. 307 
  308. 308 <code>example.com/index.php/<var>blog</var>/<samp>_utility</samp>/</code>
  309. 309

  311. 311 
  312. 312 <a name="subfolders"></a>
  313. 313 <h2>Organizing Your Controllers into Sub-folders</h2>
  314. 314 
  315. 315 <p>If you are building a large application you might find it convenient to organize your controllers into sub-folders.  CodeIgniter permits you to do this.</p>
  316. 316 
  317. 317 <p>Simply create folders within your <dfn>application/controllers</dfn> directory and place your controller classes within them.</p>
  318. 318 
  319. 319 <p><strong>Note:</strong>&nbsp; When using this feature the first segment of your URI must specify the folder.  For example, lets say you have a controller
  320. 320 located here:</p>
  321. 321 
  322. 322 <code>application/controllers/<kbd>products</kbd>/shoes.php</code>
  323. 323 
  324. 324 <p>To call the above controller your URI will look something like this:</p>
  325. 325 
  326. 326 <code>example.com/index.php/products/shoes/show/123</code>
  327. 327 
  328. 328 <p>Each of your sub-folders may contain a default controller which will be
  329. 329 called if the URL contains only the sub-folder.  Simply name your default controller as specified in your
  330. 330 <dfn>application/config/routes.php</dfn> file</p>
  331. 331 
  332. 332 
  333. 333 <p>CodeIgniter also permits you to remap your URIs using its <a href="routing.html">URI Routing</a> feature.</p>
  334. 334 
  335. 335 
  336. 336 <h2><a name="constructors"></a>Class Constructors</h2>
  337. 337 
  338. 338 
  339. 339 <p>If you intend to use a constructor in any of your Controllers, you <strong>MUST</strong> place the following line of code in it:</p>
  340. 340 
  341. 341 <code>parent::__construct();</code>
  342. 342 
  343. 343 <p>The reason this line is necessary is because your local constructor will be overriding the one in the parent controller class so we need to manually call it.</p>
  344. 344 
  345. 345 <code>
  346. 346 &lt;?php<br />
  347. 347 class <kbd>Blog</kbd> extends CI_Controller {<br />
  348. 348 <br />
  349. 349 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;public function <kbd>__construct()</kbd><br />
  350. 350 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;{<br />
  351. 351 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<var>parent::__construct();</var><br />
  352. 352 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// Your own constructor code<br />
  353. 353 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;}<br />
  354. 354 }<br />
  355. 355 ?&gt;</code>
  356. 356 
  357. 357 <p>Constructors are useful if you need to set some default values, or run a default process when your class is instantiated.
  358. 358 Constructors can't return a value, but they can do some default work.</p>
  359. 359 
  360. 360 <a name="reserved"></a>
  361. 361 <h2>Reserved Function Names</h2>
  362. 362 
  363. 363 <p>Since your controller classes will extend the main application controller you
  364. 364 must be careful not to name your functions identically to the ones used by that class, otherwise your local functions
  365. 365 will override them. See <a href="reserved_names.html">Reserved Names</a> for a full list.</p>
  366. 366 
  367. 367 <h2>That's it!</h2>
  368. 368 
  369. 369 <p>That, in a nutshell, is all there is to know about controllers.</p>
