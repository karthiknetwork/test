  57.  57 
  58.  58 <h1>Helper Functions</h1>
  59.  59 
  60.  60 <p>Helpers, as the name suggests, help you with tasks.  Each helper file is simply a collection of functions in a particular
  61.  61 category.  There are <dfn>URL Helpers</dfn>, that assist in creating links, there are <dfn>Form Helpers</dfn>
  62.  62 that help you create form elements, <dfn>Text Helpers</dfn> perform various text formatting routines,
  63.  63 <dfn>Cookie Helpers</dfn> set and read cookies, <dfn>File Helpers</dfn> help you deal with files, etc.
  64.  64 </p>
  65.  65 
  66.  66 <p>Unlike most other systems in CodeIgniter, Helpers are not written in an Object Oriented format.  They are simple, procedural functions.
  67.  67 Each helper function performs one specific task, with no dependence on other functions.</p>
  68.  68 
  69.  69 <p>CodeIgniter does not load Helper Files by default, so the first step in using
  70.  70 a Helper is to load it.  Once loaded, it becomes globally available in your <a href="../general/controllers.html">controller</a> and <a href="../general/views.html">views</a>.</p>
  71.  71 
  72.  72 <p>Helpers are typically stored in your <dfn>system/helpers</dfn>, or <dfn>application/helpers </dfn>directory. CodeIgniter will look first in your <dfn>application/helpers</dfn>
  73.  73 directory.  If the directory does not exist or the specified helper is not located there CI will instead look in your global
  74.  74 <dfn>system/helpers</dfn> folder.</p>
  75.  75 
  76.  76 
  77.  77 <h2>Loading a Helper</h2>
  78.  78 
  79.  79 <p>Loading a helper file is quite simple using the following function:</p>
  80.  80 
  81.  81 <code>$this->load->helper('<var>name</var>');</code>
  82.  82 
  83.  83 <p>Where <var>name</var> is the file name of the helper, without the .php file extension or the "helper" part.</p>
  84.  84 
  85.  85 <p>For example, to load the <dfn>URL Helper</dfn> file, which is named <var>url_helper.php</var>, you would do this:</p>
  86.  86 
  87.  87 <code>$this->load->helper('<var>url</var>');</code>
  88.  88 
  89.  89 <p>A helper can be loaded anywhere within your controller functions (or even within your View files, although that's not a good practice),
  90.  90 as long as you load it before you use it.  You can load your helpers in your controller constructor so that they become available
  91.  91 automatically in any function, or you can load a helper in a specific function that needs it.</p>
  92.  92 
  93.  93 <p class="important">Note: The Helper loading function above does not return a value, so don't try to assign it to a variable.  Just use it as shown.</p>
  94.  94 
  95.  95 
  96.  96 <h2>Loading Multiple Helpers</h2>
  97.  97 
  98.  98 <p>If you need to load more than one helper you can specify them in an array, like this:</p>
  99.  99 
  100. 100 <code>$this->load->helper( <samp>array(</samp>'<var>helper1</var>', '<var>helper2</var>', '<var>helper3</var>'<samp>)</samp> );</code>
  101. 101 
  102. 102 <h2>Auto-loading Helpers</h2>
  103. 103 
  104. 104 <p>If you find that you need a particular helper globally throughout your application, you can tell CodeIgniter to auto-load it during system initialization.
  105. 105 This is done by opening the <var>application/config/autoload.php</var> file and adding the helper to the autoload array.</p>
  106. 106 
  107. 107 
  108. 108 <h2>Using a Helper</h2>
  109. 109 
  110. 110 <p>Once you've loaded the Helper File containing the function you intend to use, you'll call it the way you would a standard PHP function.</p>
  111. 111 
  112. 112 <p>For example, to create a link using the <dfn>anchor()</dfn> function in one of your view files you would do this:</p>
  113. 113 
  114. 114 <code>&lt;?php echo anchor('blog/comments', 'Click Here');?&gt;</code>
  115. 115 
  116. 116 <p>Where "Click Here" is the name of the link, and "blog/comments" is the URI to the controller/function you wish to link to.</p>
  117. 117 
  118. 118 <h2>"Extending" Helpers</h2>
  119. 119 
  120. 120 <p>To "extend" Helpers, create a file in your <dfn>application/helpers/</dfn> folder with an identical name to the existing Helper, but prefixed with <kbd>MY_</kbd> (this item is configurable.  See below.).</p>
  121. 121 
  122. 122 <p>If all you need to do is add some functionality to an existing helper - perhaps add a function or two, or change how a particular
  123. 123  helper function operates - then it's overkill to replace the entire helper with your version.  In this case it's better to simply
  124. 124  "extend" the Helper.  The term "extend" is used loosely since Helper functions are procedural and discrete and cannot be extended
  125. 125  in the traditional programmatic sense.  Under the hood, this gives you the ability to add to the functions a Helper provides,
  126. 126  or to modify how the native Helper functions operate.</p>
  127. 127 
  128. 128 <p>For example, to extend the native <kbd>Array Helper</kbd> you'll create a file named <dfn>application/helpers/</dfn><kbd>MY_array_helper.php</kbd>, and add or override functions:</p>
  129. 129 
  130. 130 <code>
  131. 131 // any_in_array() is not in the Array Helper, so it defines a new function<br />
  132. 132 function any_in_array($needle, $haystack)<br />
  133. 133 {<br />
  134. 134 &nbsp;&nbsp;&nbsp;&nbsp;$needle = (is_array($needle)) ? $needle : array($needle);<br />
  135. 135  <br />
  136. 136 &nbsp;&nbsp;&nbsp;&nbsp;foreach ($needle as $item)<br />
  137. 137 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  138. 138 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;if (in_array($item, $haystack))<br />
  139. 139 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;{<br />
  140. 140 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return TRUE;<br />
  141. 141 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;}<br />
  142. 142 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;}<br />
  143. 143  <br />
  144. 144 &nbsp;&nbsp;&nbsp;&nbsp;return FALSE;<br />
  145. 145 }<br />
  146. 146 <br />
  147. 147 // random_element() is included in Array Helper, so it overrides the native function<br />
  148. 148 function random_element($array)<br />
  149. 149 {<br />
  150. 150 &nbsp;&nbsp;&nbsp;&nbsp;shuffle($array);<br />
  151. 151 &nbsp;&nbsp;&nbsp;&nbsp;return array_pop($array);<br />
  152. 152 }<br />
  153. 153 </code>
  154. 154 
  155. 155 <h3>Setting Your Own Prefix</h3>
  156. 156 
  157. 157 <p>The filename prefix for "extending" Helpers is the same used to extend libraries and Core classes.  To set your own prefix, open your <dfn>application/config/config.php</dfn> file and look for this item:</p>
  158. 158 
  159. 159 <code>$config['subclass_prefix'] = 'MY_';</code>
  160. 160 
  161. 161 <p>Please note that all native CodeIgniter libraries are prefixed with <kbd>CI_</kbd> so DO NOT use that as your prefix.</p>
