  59.  59 <h1>Date Helper</h1>
  60.  60 
  61.  61 <p>The Date Helper file contains functions that help you work with dates.</p>
  62.  62 
  63.  63 
  64.  64 <h2>Loading this Helper</h2>
  65.  65 
  66.  66 <p>This helper is loaded using the following code:</p>
  67.  67 <code>$this->load->helper('date');</code>
  68.  68 
  69.  69 
  70.  70 <p>The following functions are available:</p>
  71.  71 
  72.  72 <h2>now()</h2>
  73.  73 
  74.  74 <p>Returns the current time as a Unix timestamp, referenced either to your server's local time or GMT, based on the "time reference"
  75.  75 setting in your config file.  If you do not intend to set your master time reference to GMT (which you'll typically do if you
  76.  76 run a site that lets each user set their own timezone settings) there is no benefit to using this function over PHP's time() function.
  77.  77 </p>
  78.  78 
  79.  79 
  80.  80 
  81.  81 
  82.  82 <h2>mdate()</h2>
  83.  83 
  84.  84 <p>This function is identical to PHPs <a href="http://www.php.net/date">date()</a> function, except that it lets you
  85.  85 use MySQL style date codes, where each code letter is preceded with a percent sign:  %Y %m %d etc.</p>
  86.  86 
  87.  87 <p>The benefit of doing dates this way is that you don't have to worry about escaping any characters that
  88.  88 are not date codes, as you would normally have to do with the date() function.  Example:</p>
  89.  89 
  90.  90 <code>$datestring = "Year: %Y Month: %m Day: %d - %h:%i %a";<br />
  91.  91 $time = time();<br />
  92.  92 <br />
  93.  93 echo mdate($datestring, $time);</code>
  94.  94 
  95.  95 <p>If a timestamp is not included in the second parameter the current time will be used.</p>
  96.  96 
  97.  97 
  98.  98 <h2>standard_date()</h2>
  99.  99 
  100. 100 <p>Lets you generate a date string in one of several standardized formats. Example:</p>
  101. 101 
  102. 102 <code>
  103. 103 $format = 'DATE_RFC822';<br />
  104. 104 $time = time();<br />
  105. 105 <br />
  106. 106 echo standard_date($format, $time);
  107. 107 </code>
  108. 108 
  109. 109 <p>The first parameter must contain the format, the second parameter must contain the date as a Unix timestamp.</p>

  170. 170 
  171. 171 <h2>local_to_gmt()</h2>
  172. 172 
  173. 173 <p>Takes a Unix timestamp as input and returns it as GMT.  Example:</p>
  174. 174 
  175. 175 <code>$now = time();<br />
  176. 176 <br />
  177. 177 $gmt = local_to_gmt($now);</code>
  178. 178 
  179. 179 
  180. 180 <h2>gmt_to_local()</h2>
  181. 181 
  182. 182 <p>Takes a Unix timestamp (referenced to GMT) as input, and converts it to a localized timestamp based on the
  183. 183 timezone and Daylight Saving time submitted.  Example:</p>
  184. 184 
  185. 185 <code>
  186. 186 $timestamp = '1140153693';<br />
  187. 187 $timezone  = 'UM8';<br />
  188. 188 $daylight_saving = TRUE;<br />
  189. 189 <br />
  190. 190 echo gmt_to_local($timestamp, $timezone, $daylight_saving);</code>
  191. 191 
  192. 192 <p><strong>Note:</strong> For a list of timezones see the reference at the bottom of this page.</p>
  193. 193 
  194. 194 <h2>mysql_to_unix()</h2>
  195. 195 
  196. 196 <p>Takes a MySQL Timestamp as input and returns it as Unix. Example:</p>
  197. 197 
  198. 198 <code>$mysql = '20061124092345';<br />
  199. 199 <br />
  200. 200 $unix = mysql_to_unix($mysql);</code>
  201. 201 
  202. 202 
  203. 203 <h2>unix_to_human()</h2>
  204. 204 
  205. 205 <p>Takes a Unix timestamp as input and returns it in a human readable format with this prototype:</p>
  206. 206 
  207. 207 <code>YYYY-MM-DD HH:MM:SS AM/PM</code>
  208. 208 
  209. 209 <p>This can be useful if you need to display a date in a form field for submission.</p>
  210. 210 
  211. 211 <p>The time can be formatted with or without seconds, and it can be set to European or US format.  If only
  212. 212 the timestamp is submitted it will return the time without seconds formatted for the U.S. Examples:</p>
  213. 213 
  214. 214 <code>$now = time();<br />
  215. 215 <br />
  216. 216 echo unix_to_human($now); // U.S. time, no seconds<br />
  217. 217 <br />
  218. 218 echo unix_to_human($now, TRUE, 'us'); // U.S. time with seconds<br />
  219. 219 <br />
  220. 220 echo unix_to_human($now, TRUE, 'eu'); // Euro time with seconds</code>
  221. 221 
  222. 222 
  223. 223 <h2>human_to_unix()</h2>
  224. 224 
  225. 225 <p>The opposite of the above function.  Takes a "human" time as input and returns it as Unix.  This function is
  226. 226 useful if you accept "human" formatted dates submitted via a form.  Returns FALSE (boolean) if
  227. 227 the date string passed to it is not formatted as indicated above.  Example:</p>
  228. 228 
  229. 229 <code>$now = time();<br />
  230. 230 <br />
  231. 231 $human = unix_to_human($now);<br />
  232. 232 <br />
  233. 233 $unix = human_to_unix($human);</code>
  234. 234 
  235. 235 
  236. 236 
  237. 237 
  238. 238 
  239. 239 <h2>timespan()</h2>
  240. 240 
  241. 241 <p>Formats a unix timestamp so that is appears similar to this:</p>
  242. 242 
  243. 243 <code>1 Year, 10 Months, 2 Weeks, 5 Days, 10 Hours, 16 Minutes</code>
  244. 244 
  245. 245 <p>The first parameter must contain a Unix timestamp.  The second parameter must contain a
  246. 246 timestamp that is greater that the first timestamp.  If the second parameter empty, the current time will be used.  The most common purpose
  247. 247 for this function is to show how much time has elapsed from some point in time in the past to now.  Example:</p>
  248. 248 
  249. 249 <code>$post_date = '1079621429';<br />
  250. 250 $now = time();<br />
  251. 251 <br />
  252. 252 echo timespan($post_date, $now);</code>
  253. 253 
  254. 254 <p class="important"><strong>Note:</strong> The text generated by this function is found in the following language file: language/&lt;your_lang&gt;/date_lang.php</p>
  255. 255 
  256. 256 
  257. 257 <h2>days_in_month()</h2>
  258. 258 
  259. 259 <p>Returns the number of days in a given month/year. Takes leap years into account.  Example:</p>
  260. 260 <code>echo days_in_month(06, 2005);</code>
  261. 261 
  262. 262 <p>If the second parameter is empty, the current year will be used.</p>
  263. 263 <h2>timezones()</h2>
  264. 264 <p> Takes a timezone reference (for a list of valid timezones, see the &quot;Timezone Reference&quot; below) and returns the number of hours offset from UTC.</p>
  265. 265 <p><code>echo timezones('UM5');</code></p>
  266. 266 <p>This function is useful when used with timezone_menu(). </p>
