  57.  57 
  58.  58 <h1>Upgrading from 1.2 to 1.3</h1>
  59.  59 
  60.  60 <p class="important"><strong>Note:</strong> The instructions on this page assume you are running version 1.2.  If you
  61.  61 have not upgraded to that version please do so first.</p>
  62.  62 
  63.  63 
  64.  64 <p>Before performing an update you should take your site offline by replacing the index.php file
  65.  65 with a static one.</p>
  66.  66 
  67.  67 
  68.  68 <h2>Step 1: Update your CodeIgniter files</h2>
  69.  69 
  70.  70 <p>Replace the following directories in your "system" folder with the new versions:</p>
  71.  71 
  72.  72 <p class="important"><strong>Note:</strong> If you have any custom developed files in these folders please make copies of them first.</p>
  73.  73 
  74.  74 <ul>
  75.  75 <li>application/<strong>models</strong>/&nbsp;&nbsp;  (new for 1.3)</li>
  76.  76 <li>codeigniter&nbsp;&nbsp;   (new for 1.3)</li>
  77.  77 <li>drivers</li>
  78.  78 <li>helpers</li>
  79.  79 <li>init</li>
  80.  80 <li>language</li>
  81.  81 <li>libraries</li>
  82.  82 <li>plugins</li>
  83.  83 <li>scaffolding</li>
  84.  84 </ul>
  85.  85 
  86.  86 
  87.  87 <h2>Step 2: Update your error files</h2>
  88.  88 
  89.  89 <p>Version 1.3 contains two new error templates located in <dfn>application/errors</dfn>, and for naming consistency the other error templates have
  90.  90 been renamed.</p>
  91.  91 
  92.  92 <p>If you <strong>have not</strong> customized any of the error templates simply
  93.  93 replace this folder:</p>
  94.  94 
  95.  95 <ul>
  96.  96 <li>application/errors/</li>
  97.  97 </ul>
  98.  98 
  99.  99 <p>If you <strong>have</strong> customized your error templates, rename them as follows:</p>
  100. 100 
  101. 101 
  102. 102 <ul>
  103. 103 <li>404.php&nbsp;&nbsp; =&nbsp; error_404.php</li>
  104. 104 <li>error.php&nbsp;&nbsp; =&nbsp; error_general.php</li>
  105. 105 <li>error_db.php&nbsp;&nbsp; (new)</li>
  106. 106 <li>error_php.php&nbsp;&nbsp; (new)</li>
  107. 107 </ul>
  108. 108 
  109. 109 
  110. 110 <h2>Step 3: Update your index.php file</h2>
  111. 111 
  112. 112 <p>Please open your main <dfn>index.php</dfn> file (located at your root).  At the very bottom of the file, change this:</p>
  113. 113 
  114. 114 <code>require_once BASEPATH.'libraries/Front_controller'.EXT;</code>
  115. 115 
  116. 116 <p>To this:</p>
  117. 117 
  118. 118 <code>require_once BASEPATH.'codeigniter/CodeIgniter'.EXT;</code>
  119. 119 
  120. 120 
  121. 121 <h2>Step 4: Update your config.php file</h2>
  122. 122 
  123. 123 <p>Open your <dfn>application/config/config.php</dfn> file and add these new items:</p>
  124. 124 
  125. 125 <pre>
  126. 126 /*
  127. 127 |------------------------------------------------
  128. 128 | URL suffix
  129. 129 |------------------------------------------------
  130. 130 |
  131. 131 | This option allows you to add a suffix to all URLs.
  132. 132 | For example, if a URL is this:
  133. 133 |
  134. 134 | example.com/index.php/products/view/shoes
  135. 135 |
  136. 136 | You can optionally add a suffix, like ".html",
  137. 137 | making the page appear to be of a certain type:
  138. 138 |
  139. 139 | example.com/index.php/products/view/shoes.html
  140. 140 |
  141. 141 */
  142. 142 $config['url_suffix'] = "";
  143. 143 
  144. 144 
  145. 145 /*
  146. 146 |------------------------------------------------
  147. 147 | Enable Query Strings
  148. 148 |------------------------------------------------
  149. 149 |
  150. 150 | By default CodeIgniter uses search-engine and
  151. 151 | human-friendly segment based URLs:
  152. 152 |
  153. 153 | example.com/who/what/where/
  154. 154 |
  155. 155 | You can optionally enable standard query string
  156. 156 | based URLs:
  157. 157 |
  158. 158 | example.com?who=me&what=something&where=here
  159. 159 |
  160. 160 | Options are: TRUE or FALSE (boolean)
  161. 161 |
  162. 162 | The two other items let you set the query string "words"
  163. 163 | that will invoke your controllers and functions:
  164. 164 | example.com/index.php?c=controller&m=function
  165. 165 |
  166. 166 */
  167. 167 $config['enable_query_strings'] = FALSE;
  168. 168 $config['controller_trigger'] = 'c';
  169. 169 $config['function_trigger'] = 'm';
  170. 170 </pre>
  171. 171 
  172. 172 
  173. 173 <h2>Step 5: Update your database.php file</h2>
  174. 174 
  175. 175 <p>Open your <dfn>application/config/database.php</dfn> file and add these new items:</p>
  176. 176 
  177. 177 <pre>
  178. 178 $db['default']['dbprefix'] = "";
  179. 179 $db['default']['active_r'] = TRUE;
  180. 180 </pre>
  181. 181 
  182. 182 
  183. 183 <h2>Step 6: Update your user guide</h2>
  184. 184 
  185. 185 <p>Please also replace your local copy of the user guide with the new version.</p>
