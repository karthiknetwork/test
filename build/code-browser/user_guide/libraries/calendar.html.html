  61.  61 <h1>Calendaring Class</h1>
  62.  62 
  63.  63 <p>The Calendar class enables you to dynamically create calendars. Your calendars can be formatted through the use of a calendar
  64.  64 template, allowing 100% control over every aspect of its design. In addition, you can pass data to your calendar cells.</p>
  65.  65 
  66.  66 <h2>Initializing the Class</h2>
  67.  67 
  68.  68 <p>Like most other classes in CodeIgniter, the Calendar class is initialized in your controller using the <dfn>$this->load->library</dfn> function:</p>
  69.  69 
  70.  70 <code>$this->load->library('calendar');</code>
  71.  71 <p>Once loaded, the Calendar object will be available using: <dfn>$this->calendar</dfn></p>
  72.  72 
  73.  73 
  74.  74 <h2>Displaying a Calendar</h2>
  75.  75 
  76.  76 <p>Here is a very simple example showing how you can display a calendar:</p>
  77.  77 
  78.  78 <code>$this->load->library('calendar');<br />
  79.  79 <br />
  80.  80 echo $this->calendar->generate();</code>
  81.  81 
  82.  82 <p>The above code will generate a calendar for the current month/year based on your server time.
  83.  83 To show a calendar for a specific month and year you will pass this information to the calendar generating function:</p>
  84.  84 
  85.  85 <code>$this->load->library('calendar');<br />
  86.  86 <br />
  87.  87 echo $this->calendar->generate(<kbd>2006</kbd>, <kbd>6</kbd>);</code>
  88.  88 
  89.  89 <p>The above code will generate a calendar showing the month of June in 2006.  The first parameter specifies the year, the second parameter specifies the month.</p>
  90.  90 
  91.  91 <h2>Passing Data to your Calendar Cells</h2>
  92.  92 
  93.  93 <p>To add data to your calendar cells involves creating an associative array in which the keys correspond to the days
  94.  94 you wish to populate and the array value contains the data.  The array is passed to the third parameter of the calendar
  95.  95 generating function.  Consider this example:</p>
  96.  96 
  97.  97 <code>$this->load->library('calendar');<br />
  98.  98 <br />
  99.  99 $data = array(<br />
  100. 100 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3&nbsp; => 'http://example.com/news/article/2006/03/',<br />
  101. 101 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;7&nbsp; => 'http://example.com/news/article/2006/07/',<br />
  102. 102 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;13 => 'http://example.com/news/article/2006/13/',<br />
  103. 103 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;26 => 'http://example.com/news/article/2006/26/'<br />
  104. 104 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;);<br />
  105. 105 <br />
  106. 106 echo $this->calendar->generate(<kbd>2006</kbd>, <kbd>6</kbd>, <var>$data</var>);</code>
  107. 107 
  108. 108 <p>Using the above example, day numbers 3, 7, 13, and 26 will become links pointing to the URLs you've provided.</p>
  109. 109 
  110. 110 <p class="important"><strong>Note:</strong> By default it is assumed that your array will contain links.
  111. 111 In the section that explains the calendar template below you'll see how you can customize
  112. 112 how data passed to your cells is handled so you can pass different types of information.</p>
  113. 113 
  114. 114 
  115. 115 <h2>Setting Display Preferences</h2>
  116. 116 
  117. 117 <p>There are seven preferences you can set to control various aspects of the calendar.  Preferences are set by passing an
  118. 118 array of preferences in the second parameter of the loading function. Here is an example:</p>
  119. 119 
  120. 120 
  121. 121 <code>
  122. 122 $prefs = array (<br />
  123. 123 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'start_day'&nbsp;&nbsp;&nbsp; => 'saturday',<br />
  124. 124 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'month_type'&nbsp;&nbsp; => 'long',<br />
  125. 125 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'day_type'&nbsp;&nbsp;&nbsp;&nbsp; => 'short'<br />
  126. 126 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;);<br />
  127. 127 <br />
  128. 128 $this->load->library('calendar', $prefs);<br />
  129. 129 <br />
  130. 130 echo $this->calendar->generate();</code>
  131. 131 
  132. 132 <p>The above code would start the calendar on saturday, use the "long" month heading, and the "short" day names.  More information
  133. 133 regarding preferences below.</p>
  134. 134 
  135. 135 
  136. 136 
  137. 137 
  138. 138 <table cellpadding="0" cellspacing="1" border="0" style="width:100%" class="tableborder">

  146. 146 </tr><tr>
  147. 147 <td class="td"><strong>local_time</strong></td><td class="td">time()</td><td class="td">None</td><td class="td">A Unix timestamp corresponding to the current time.</td>
  148. 148 </tr><tr>
  149. 149 <td class="td"><strong>start_day</strong></td><td class="td">sunday</td><td class="td">Any week day (sunday, monday, tuesday, etc.)</td><td class="td">Sets the day of the week the calendar should start on.</td>

  156. 156 </tr><tr>
  157. 157 <td class="td"><strong>next_prev_url</strong></td><td class="td">None</td><td class="td">A URL</td><td class="td">Sets the basepath used in the next/previous calendar links.</td>
  158. 158 </tr>

  162. 162 
  163. 163 <h2>Showing Next/Previous Month Links</h2>
  164. 164 
  165. 165 <p>To allow your calendar to dynamically increment/decrement via the next/previous links requires that you set up your calendar
  166. 166 code similar to this example:</p>
  167. 167 
  168. 168 
  169. 169 <code>$prefs = array (<br />
  170. 170 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'show_next_prev'&nbsp;&nbsp;=> TRUE,<br />
  171. 171 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'next_prev_url'&nbsp;&nbsp; => 'http://example.com/index.php/calendar/show/'<br />
  172. 172 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;);<br />
  173. 173 <br />
  174. 174 $this-&gt;load-&gt;library('calendar', $prefs);<br />
  175. 175 <br />
  176. 176 echo $this->calendar->generate(<var>$this->uri->segment(3)</var>, <var>$this->uri->segment(4)</var>);</code>
  177. 177 
  178. 178 <p>You'll notice a few things about the above example:</p>
  179. 179 
  180. 180 <ul>
  181. 181 <li>You must set the "show_next_prev" to TRUE.</li>
  182. 182 <li>You must supply the URL to the controller containing your calendar in the "next_prev_url" preference.</li>
  183. 183 <li>You must supply the "year" and "month" to the calendar generating function via the URI segments where they appear (Note:  The calendar class automatically adds the year/month to the base URL you provide.).</li>
  184. 184 </ul>
  185. 185

  189. 189 
  190. 190 <p>By creating a calendar template you have 100% control over the design of your calendar. Each component of your
  191. 191 calendar will be placed within a pair of pseudo-variables as shown here:</p>
  192. 192 
  193. 193 
  194. 194 <code>
  195. 195 $prefs['template'] = '<br /><br />
  196. 196 &nbsp;&nbsp;&nbsp;<dfn>{table_open}</dfn><var>&lt;table border="0" cellpadding="0" cellspacing="0"></var><dfn>{/table_open}</dfn><br />

  205. 205 <br />
  206. 206 &nbsp;&nbsp;&nbsp;<dfn>{week_row_start}</dfn><var>&lt;tr></var><dfn>{/week_row_start}</dfn><br />
  207. 207 &nbsp;&nbsp;&nbsp;<dfn>{week_day_cell}</dfn><var>&lt;td></var><dfn>{week_day}</dfn><var>&lt;/td></var><dfn>{/week_day_cell}</dfn><br />
  208. 208 &nbsp;&nbsp;&nbsp;<dfn>{week_row_end}</dfn><var>&lt;/tr></var><dfn>{/week_row_end}</dfn><br />
  209. 209 <br />
  210. 210 &nbsp;&nbsp;&nbsp;<dfn>{cal_row_start}</dfn><var>&lt;tr></var><dfn>{/cal_row_start}</dfn><br />
