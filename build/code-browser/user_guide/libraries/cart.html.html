  59.  59 <h1>Shopping Cart Class</h1>
  60.  60 
  61.  61 <p>The Cart Class permits items to be added to a session that stays active while a user is browsing your site.
  62.  62 These items can be retrieved and displayed in a standard "shopping cart" format, allowing the user to update the quantity or remove items from the cart.</p>
  63.  63 
  64.  64 <p>Please note that the Cart Class ONLY provides the core "cart" functionality. It does not provide shipping, credit card authorization, or other processing components.</p>
  65.  65 
  66.  66 
  67.  67 <h2>Initializing the Shopping Cart Class</h2>
  68.  68 
  69.  69 <p><strong>Important:</strong> The Cart class utilizes CodeIgniter's
  70.  70 <a href="sessions.html">Session Class</a>  to save the cart information to a database, so before using the Cart class you must set up a database table
  71.  71 as indicated in the <a href="sessions.html">Session Documentation</a> , and set the session preferences in your <kbd>application/config/config.php</kbd> file to utilize a database.</p>
  72.  72 
  73.  73 <p>To initialize the Shopping Cart Class in your controller constructor, use the <dfn>$this->load->library</dfn> function:</p>
  74.  74 
  75.  75 <code>$this->load->library('cart');</code>
  76.  76 <p>Once loaded, the Cart object will be available using: <dfn>$this->cart</dfn></p>
  77.  77 
  78.  78 <p class="important"><strong>Note:</strong> The Cart Class will load and initialize the Session Class automatically, so unless you are using sessions elsewhere in your application, you do not need to load the Session class.</p>
  79.  79 
  80.  80 <h2>Adding an Item to The Cart</h2>
  81.  81 
  82.  82 <p>To add an item to the shopping cart, simply pass an array with the product information to the <dfn>$this->cart->insert()</dfn> function, as shown below:</p>
  83.  83 
  84.  84 <code>
  85.  85 $data = array(<br />
  86.  86 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'id'&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; => 'sku_123ABC',<br />
  87.  87 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'qty'&nbsp;&nbsp;&nbsp;&nbsp; => 1,<br />
  88.  88 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'price'&nbsp;&nbsp; => 39.95,<br />
  89.  89 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'name'&nbsp;&nbsp;&nbsp; => 'T-Shirt',<br />
  90.  90 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'options' => array('Size' => 'L', 'Color' => 'Red')<br />
  91.  91 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;);<br />
  92.  92 <br />
  93.  93 
  94.  94 $this->cart->insert($data);
  95.  95 
  96.  96 </code>
  97.  97 
  98.  98 <p class="important"><strong>Important:</strong> The first four array indexes above (<dfn>id</dfn>, <dfn>qty</dfn>, <dfn>price</dfn>, and <dfn>name</dfn>) are <strong>required</strong>.
  99.  99 If you omit any of them the data will not be saved to the cart. The fifth index (<dfn>options</dfn>) is optional.
  100. 100 It is intended to be used in cases where your product has options associated with it. Use an array for options, as shown above.</p>
  101. 101 
  102. 102 <p>The five reserved indexes are:</p>
  103. 103 
  104. 104 <ul>
  105. 105 <li><strong>id</strong> - Each product in your store must have a unique identifier. Typically this will be an "sku" or other such identifier.</li>
  106. 106 <li><strong>qty</strong> - The quantity being purchased.
  107. 107 <li><strong>price</strong> - The price of the item.
  108. 108 <li><strong>name</strong> - The name of the item.
  109. 109 <li><strong>options</strong> - Any additional attributes that are needed to identify the product. These must be passed via an array.
  110. 110 </ul>
  111. 111 
  112. 112 <p>In addition to the five indexes above, there are two reserved words: <dfn>rowid</dfn> and <dfn>subtotal</dfn>.  These are used internally by the Cart class, so
  113. 113 please do NOT use those words as index names when inserting data into the cart.</p>
  114. 114 
  115. 115 <p>Your array may contain additional data. Anything you include in your array will be stored in the session. However, it is best to standardize your data among all your products in order to make displaying the information in a table easier.</p>
  116. 116 
  117. 117 <p>The insert() method will return the $rowid if you successfully insert a single item.</p>

  267. 267 
  268. 268 <p><strong>What is a Row ID?</strong>&nbsp; The <kbd>row ID</kbd> is a unique identifier that is generated by the cart code when an item is added to the cart.  The reason a
  269. 269 unique ID is created is so that identical products with different options can be managed by the cart.</p>
  270. 270 
  271. 271 <p>For example, let's say someone buys two identical t-shirts (same product ID), but in different sizes. The product ID (and other attributes) will be
  272. 272 identical for both sizes because it's the same shirt. The only difference will be the size. The cart must therefore have a means of identifying this
  273. 273 difference so that the two sizes of shirts can be managed independently. It does so by creating a unique "row ID" based on the product ID and any options associated with it.</p>
  274. 274 
  275. 275 <p>In nearly all cases, updating the cart will be something the user does via the "view cart" page, so as a developer, it is unlikely that you will ever have to concern yourself
  276. 276 with the "row ID", other then making sure your "view cart" page contains this information in a hidden form field, and making sure it gets passed to the update
  277. 277 function when the update form is submitted. Please examine the construction of the "view cart" page above for more information.</p>

  305. 305 
  306. 306 <h2>$this->cart->contents();</h2>
