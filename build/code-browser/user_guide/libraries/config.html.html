  59.  59 <h1>Config Class</h1>
  60.  60 
  61.  61 <p>The Config class provides a means to retrieve configuration preferences.  These preferences can
  62.  62 come from the default config file (<samp>application/config/config.php</samp>) or from your own custom config files.</p>
  63.  63 
  64.  64 <p class="important"><strong>Note:</strong> This class is initialized automatically by the system so there is no need to do it manually.</p>
  65.  65 
  66.  66 
  67.  67 <h2>Anatomy of a Config File</h2>
  68.  68 
  69.  69 <p>By default, CodeIgniter has one primary config file, located at <samp>application/config/config.php</samp>.  If you open the file using
  70.  70 your text editor you'll see that config items are stored in an array called <var>$config</var>.</p>
  71.  71 
  72.  72 <p>You can add your own config items to
  73.  73 this file, or if you prefer to keep your configuration items separate (assuming you even need config items),
  74.  74 simply create your own file and save it in <dfn>config</dfn> folder.</p>
  75.  75 
  76.  76 <p><strong>Note:</strong> If you do create your own config files use the same format as the primary one, storing your items in
  77.  77 an array called  <var>$config</var>. CodeIgniter will intelligently manage these files so there will be no conflict even though
  78.  78 the array has the same name (assuming an array index is not named the same as another).</p>
  79.  79 
  80.  80 <h2>Loading a Config File</h2>
  81.  81 
  82.  82 <p><strong>Note:</strong> CodeIgniter automatically loads the primary config file (<samp>application/config/config.php</samp>),
  83.  83 so you will only need to load a config file if you have created your own.</p>
  84.  84 
  85.  85 <p>There are two ways to load a config file:</p>
  86.  86 
  87.  87 <ol><li><strong>Manual Loading</strong>
  88.  88 
  89.  89 <p>To load one of your custom config files you will use the following function within the <a href="../general/controllers.html">controller</a> that needs it:</p>
  90.  90 
  91.  91 <code>$this->config->load('<var>filename</var>');</code>
  92.  92 
  93.  93 <p>Where <var>filename</var> is the name of your config file, without the .php file extension.</p>
  94.  94 
  95.  95 <p>If you need to load multiple config files normally they will be merged into one master config array.  Name collisions can occur, however, if
  96.  96 you have identically named array indexes in different config files.  To avoid collisions you can set the second parameter to <kbd>TRUE</kbd>
  97.  97 and each config file will be stored in an array index corresponding to the name of the config file. Example:</p>
  98.  98 
  99.  99 <code>
  100. 100 // Stored in an array with this prototype:  $this->config['blog_settings'] = $config<br />
  101. 101 $this->config->load('<var>blog_settings</var>', <kbd>TRUE</kbd>);</code>
  102. 102 
  103. 103 <p>Please see the section entitled <dfn>Fetching Config Items</dfn> below to learn how to retrieve config items set this way.</p>
  104. 104 
  105. 105 <p>The third parameter allows you to suppress errors in the event that a config file does not exist:</p>
  106. 106 
  107. 107 <code>$this->config->load('<var>blog_settings</var>', <dfn>FALSE</dfn>, <kbd>TRUE</kbd>);</code>
  108. 108 
  109. 109 </li>
  110. 110 <li><strong>Auto-loading</strong>
  111. 111 
  112. 112 <p>If you find that you need a particular config file globally, you can have it loaded automatically by the system.  To do this,
  113. 113 open the <strong>autoload.php</strong> file, located at <samp>application/config/autoload.php</samp>, and add your config file as
  114. 114 indicated in the file.</p>
  115. 115 </li>
  116. 116 </ol>
  117. 117 
  118. 118 
  119. 119 <h2>Fetching Config Items</h2>
  120. 120 
  121. 121 <p>To retrieve an item from your config file, use the following function:</p>
  122. 122 
  123. 123 <code>$this->config->item('<var>item name</var>');</code>
  124. 124 
  125. 125 <p>Where <var>item name</var> is the $config array index you want to retrieve. For example, to fetch your language choice you'll do this:</p>
  126. 126 
  127. 127 <code>$lang = $this->config->item('language');</code>
  128. 128 
  129. 129 <p>The function returns FALSE (boolean) if the item you are trying to fetch does not exist.</p>
  130. 130 
  131. 131 <p>If you are using the second parameter of the <kbd>$this->config->load</kbd> function in order to assign your config items to a specific index
  132. 132 you can retrieve it by specifying the index name in the second parameter of the <kbd>$this->config->item()</kbd> function.  Example:</p>
  133. 133 
  134. 134 <code>
  135. 135 // Loads a config file named blog_settings.php and assigns it to an index named "blog_settings"<br />
  136. 136 $this->config->load('<var>blog_settings</var>', <kbd>TRUE</kbd>);<br /><br />
  137. 137 
  138. 138 // Retrieve a config item named site_name contained within the blog_settings array<br />
  139. 139 $site_name = $this->config->item('<dfn>site_name</dfn>', '<var>blog_settings</var>');<br /><br />
  140. 140 
  141. 141 // An alternate way to specify the same item:<br />
  142. 142 $blog_config = $this->config->item('<var>blog_settings</var>');<br />
  143. 143 $site_name = $blog_config['site_name'];</code>
  144. 144 
  145. 145 <h2>Setting a Config Item</h2>
  146. 146 
  147. 147 <p>If you would like to dynamically set a config item or change an existing one, you can do so using:</p>
  148. 148 
  149. 149 <code>$this->config->set_item('<var>item_name</var>', '<var>item_value</var>');</code>
  150. 150 
  151. 151 <p>Where <var>item_name</var> is the $config array index you want to change, and <var>item_value</var> is its value.</p>
  152. 152 
  153. 153 <a name="environments"></a>
  154. 154 <h2>Environments</h2>
  155. 155 
  156. 156 <p>
  157. 157     You may load different configuration files depending on the current environment.
  158. 158     The <kbd>ENVIRONMENT</kbd> constant is defined in index.php, and is described
  159. 159     in detail in the <a href="../general/environments.html">Handling Environments</a>
  160. 160     section.
  161. 161 </p>
  162. 162 
  163. 163 <p>
  164. 164     To create an environment-specific configuration file,
  165. 165     create or copy a configuration file in application/config/{ENVIRONMENT}/{FILENAME}.php
  166. 166 </p>
  167. 167 
  168. 168 <p>For example, to create a production-only config.php, you would:</p>
  169. 169 
  170. 170 <ol>
  171. 171     <li>Create the directory application/config/production/</li>
  172. 172     <li>Copy your existing config.php into the above directory</li>
  173. 173     <li>Edit application/config/production/config.php so it contains your production settings</li>
  174. 174 </ol>
  175. 175 
  176. 176 <p>
  177. 177     When you set the <kbd>ENVIRONMENT</kbd> constant to 'production', the settings
  178. 178     for your new production-only config.php will be loaded.
  179. 179 </p>
  180. 180 
  181. 181 <p>You can place the following configuration files in environment-specific folders:</p>
  182. 182 
  183. 183 <ul>
  184. 184 <li>Default CodeIgniter configuration files</li>
  185. 185 <li>Your own custom configuration files</li>
  186. 186 </ul>
  187. 187 
  188. 188 <p><strong>Note:</strong> CodeIgniter always tries to load the configuration files for the current environment first. If the file does not exist, the global config file (i.e., the one in <samp>application/config/</samp>) is loaded. This means you are not obligated to place <strong>all</strong> of your configuration files in an environment folder &minus; only the files that change per environment.</p>
  189. 189 
  190. 190 <h2>Helper Functions</h2>
  191. 191 
  192. 192 <p>The config class has the following helper functions:</p>
  193. 193 
  194. 194 <h2>$this->config->site_url();</h2>
  195. 195 <p>This function retrieves the URL to your site, along with the "index" value you've specified in the config file.</p>
  196. 196 
  197. 197 <h2>$this->config->base_url();</h2>
  198. 198 <p>This function retrieves the URL to your site, plus an optional path such as to a stylesheet or image.</p>
  199. 199 
  200. 200 <p>The two functions above are normally accessed via the corresponding functions in the <a href="../helpers/url_helper.html">URL Helper.</a></p>
  201. 201 
  202. 202 <h2>$this->config->system_url();</h2>
  203. 203 <p>This function retrieves the URL to your <dfn>system folder</dfn>.</p>
