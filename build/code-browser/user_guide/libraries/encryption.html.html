  59.  59 <h1>Encryption Class</h1>
  60.  60 
  61.  61 <p>The Encryption Class provides two-way data encryption.  It uses a scheme that either compiles
  62.  62 the message using a randomly hashed bitwise XOR encoding scheme, or is encrypted using
  63.  63 the Mcrypt library.  If Mcrypt is not available on your server the encoded message will
  64.  64 still provide a reasonable degree of security for encrypted sessions or other such "light" purposes.
  65.  65 If Mcrypt is available, you'll be provided with a high degree of security appropriate for storage.</p>
  66.  66 
  67.  67 
  68.  68 <h2>Setting your Key</h2>
  69.  69 
  70.  70 <p>A <em>key</em> is a piece of information that controls the cryptographic process and permits an encrypted string to be decoded.
  71.  71 In fact, the key you chose will provide the <strong>only</strong> means to decode data that was encrypted with that key,
  72.  72 so not only must you choose the key carefully, you must never change it if you intend use it for persistent data.</p>
  73.  73 
  74.  74 <p>It goes without saying that you should guard your key carefully.
  75.  75 Should someone gain access to your key, the data will be easily decoded.  If your server is not totally under your control
  76.  76 it's impossible to ensure key security so you may want to think carefully before using it for anything
  77.  77 that requires high security, like storing credit card numbers.</p>
  78.  78 
  79.  79 <p>To take maximum advantage of the encryption algorithm, your key should be 32 characters in length (128 bits).
  80.  80 The key should be as random a string as you can concoct, with numbers and uppercase and lowercase letters.
  81.  81 Your key should <strong>not</strong> be a simple text string. In order to be cryptographically secure it
  82.  82 needs to be as random as possible.</p>
  83.  83 
  84.  84 <p>Your key can be either stored in your <dfn>application/config/config.php</dfn>, or you can design your own
  85.  85 storage mechanism and pass the key dynamically when encoding/decoding.</p>
  86.  86 
  87.  87 <p>To save your key to your <dfn>application/config/config.php</dfn>, open the file and set:</p>
  88.  88 <code>$config['encryption_key'] = "YOUR KEY";</code>
  89.  89 
  90.  90 
  91.  91 <h2>Message Length</h2>
  92.  92 
  93.  93 <p>It's important for you to know that the encoded messages the encryption function generates will be approximately 2.6 times longer than the original
  94.  94 message.  For example, if you encrypt the string "my super secret data", which is 21 characters in length, you'll end up
  95.  95 with an encoded string that is roughly 55 characters (we say "roughly" because the encoded string length increments in
  96.  96 64 bit clusters, so it's not exactly linear).  Keep this information in mind when selecting your data storage mechanism.  Cookies,
  97.  97 for example, can only hold 4K of information.</p>
  98.  98 
  99.  99 
  100. 100 <h2>Initializing the Class</h2>
  101. 101 
  102. 102 <p>Like most other classes in CodeIgniter, the Encryption class is initialized in your controller using the <dfn>$this->load->library</dfn> function:</p>
  103. 103 
  104. 104 <code>$this->load->library('encrypt');</code>
  105. 105 <p>Once loaded, the Encrypt library object will be available using: <dfn>$this->encrypt</dfn></p>
  106. 106 
  107. 107 
  108. 108 <h2>$this->encrypt->encode()</h2>
  109. 109 
  110. 110 <p>Performs the data encryption and returns it as a string. Example:</p>
  111. 111 <code>
  112. 112 $msg = 'My secret message';<br />
  113. 113 <br />
  114. 114 $encrypted_string = $this->encrypt->encode($msg);</code>
  115. 115 
  116. 116 <p>You can optionally pass your encryption key via the second parameter if you don't want to use the one in your config file:</p>
  117. 117 
  118. 118 <code>
  119. 119 $msg = 'My secret message';<br />
  120. 120 $key = 'super-secret-key';<br />
  121. 121 <br />
  122. 122 $encrypted_string = $this->encrypt->encode($msg, $key);</code>
  123. 123 
  124. 124 
  125. 125 <h2>$this->encrypt->decode()</h2>
  126. 126 
  127. 127 <p>Decrypts an encoded string.  Example:</p>
  128. 128 
  129. 129 <code>
  130. 130 $encrypted_string = 'APANtByIGI1BpVXZTJgcsAG8GZl8pdwwa84';<br />
  131. 131 <br />
  132. 132 $plaintext_string = $this->encrypt->decode($encrypted_string);</code>
  133. 133 
  134. 134 <p>You can optionally pass your encryption key via the second parameter if you don't want to use the one in your config file:</p>
  135. 135 
  136. 136 <code>
  137. 137 $msg = 'My secret message';<br />
  138. 138 $key = 'super-secret-key';<br />
  139. 139 <br />
  140. 140 $encrypted_string = $this->encrypt->decode($msg, $key);</code>
  141. 141 
  142. 142 
  143. 143 <h2>$this->encrypt->set_cipher();</h2>
  144. 144 
  145. 145 <p>Permits you to set an Mcrypt cipher.  By default it uses <samp>MCRYPT_RIJNDAEL_256</samp>.  Example:</p>
  146. 146 <code>$this->encrypt->set_cipher(MCRYPT_BLOWFISH);</code>
  147. 147 <p>Please visit php.net for a list of  <a href="http://php.net/mcrypt">available ciphers</a>.</p>
  148. 148 
  149. 149 <p>If you'd like to manually test whether your server supports Mcrypt you can use:</p>
  150. 150 <code>echo ( ! function_exists('mcrypt_encrypt')) ? 'Nope' : 'Yup';</code>
  151. 151 
  152. 152 
  153. 153 <h2>$this->encrypt->set_mode();</h2>
  154. 154 
  155. 155 <p>Permits you to set an Mcrypt mode.  By default it uses <samp>MCRYPT_MODE_CBC</samp>.  Example:</p>
  156. 156 <code>$this->encrypt->set_mode(MCRYPT_MODE_CFB);</code>
  157. 157 <p>Please visit php.net for a list of  <a href="http://php.net/mcrypt">available modes</a>.</p>
  158. 158 
  159. 159 
  160. 160 <h2>$this->encrypt->sha1();</h2>
  161. 161 <p>SHA1 encoding function.  Provide a string and it will return a 160 bit one way hash.  Note:  SHA1, just like MD5 is non-decodable. Example:</p>
  162. 162 <code>$hash = $this->encrypt->sha1('Some string');</code>
  163. 163 
  164. 164 <p>Many PHP installations have SHA1 support by default so if all you need is to encode a hash it's simpler to use the native
  165. 165 function:</p>
  166. 166 
  167. 167 <code>$hash = sha1('Some string');</code>
  168. 168 
  169. 169 <p>If your server does not support SHA1 you can use the provided function.</p>
  170. 170 
  171. 171 <h2 id="legacy">$this->encrypt->encode_from_legacy(<kbd>$orig_data</kbd>, <kbd>$legacy_mode</kbd> = MCRYPT_MODE_ECB, <kbd>$key</kbd> = '');</h2>
  172. 172 <p>Enables you to re-encode data that was originally encrypted with CodeIgniter 1.x to be compatible with the Encryption library in CodeIgniter 2.x.  It is only
  173. 173  necessary to use this method if you have encrypted data stored permanently such as in a file or database and are on a server that supports Mcrypt.  "Light" use encryption
  174. 174  such as encrypted session data or transitory encrypted flashdata require no intervention on your part.  However, existing encrypted Sessions will be
  175. 175  destroyed since data encrypted prior to 2.x will not be decoded.</p>
  176. 176 
  177. 177 <p class="important"><strong>Why only a method to re-encode the data instead of maintaining legacy methods for both encoding and decoding?</strong>  The algorithms in
  178. 178  the Encryption library have improved in CodeIgniter 2.x both for performance and security, and we do not wish to encourage continued use of the older methods.
  179. 179  You can of course extend the Encryption library if you wish and replace the new methods with the old and retain seamless compatibility with CodeIgniter 1.x
  180. 180  encrypted data, but this a decision that a developer should make cautiously and deliberately, if at all.</p>
  181. 181 
  182. 182 <code>$new_data = $this->encrypt->encode_from_legacy(<kbd>$old_encrypted_string</kbd>);</code>
