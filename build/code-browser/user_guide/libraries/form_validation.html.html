  98.   98 <h1>Overview</h1>
  99.   99 
  100.  100 
  101.  101 <p>Before explaining CodeIgniter's approach to data validation, let's describe the ideal scenario:</p>
  102.  102 
  103.  103 <ol>
  104.  104 <li>A form is displayed.</li>
  105.  105 <li>You fill it in and submit it.</li>
  106.  106 <li>If you submitted something invalid, or perhaps missed a required item, the form is redisplayed containing your data
  107.  107 along with an error message describing the problem.</li>
  108.  108 <li>This process continues until you have submitted a valid form.</li>
  109.  109 </ol>
  110.  110 
  111.  111 <p>On the receiving end, the script must:</p>
  112.  112 
  113.  113 <ol>
  114.  114 <li>Check for required data.</li>
  115.  115 <li>Verify that the data is of the correct type, and meets the correct criteria. For example, if a username is submitted
  116.  116 it must be validated to contain only permitted characters.  It must be of a minimum length,
  117.  117 and not exceed a maximum length. The username can't be someone else's existing username, or perhaps even a reserved word. Etc.</li>
  118.  118 <li>Sanitize the data for security.</li>
  119.  119 <li>Pre-format the data if needed (Does the data need to be trimmed?  HTML encoded?  Etc.)</li>
  120.  120 <li>Prep the data for insertion in the database.</li>
  121.  121 </ol>
  122.  122 
  123.  123 
  124.  124 <p>Although there is nothing terribly complex about the above process, it usually requires a significant
  125.  125 amount of code, and to display error messages, various control structures are usually placed within the form HTML.
  126.  126 Form validation, while simple to create, is generally very messy and tedious to implement.</p>
  127.  127 
  128.  128 <p>&nbsp;</p>

  132.  132 <h1>Form Validation Tutorial</h1>
  133.  133 
  134.  134 <p>What follows is a "hands on" tutorial for implementing CodeIgniters Form Validation.</p>
  135.  135 
  136.  136 
  137.  137 <p>In order to implement form validation you'll need three things:</p>
  138.  138 
  139.  139 <ol>
  140.  140 <li>A <a href="../general/views.html">View</a> file containing a form.</li>
  141.  141 <li>A View file containing a "success" message to be displayed upon successful submission.</li>
  142.  142 <li>A <a href="../general/controllers.html">controller</a> function to receive and process the submitted data.</li>
  143.  143 </ol>
  144.  144

  245.  245 <h2>Try it!</h2>
  246.  246 
  247.  247 <p>To try your form, visit your site using a URL similar to this one:</p>
  248.  248 
  249.  249 <code>example.com/index.php/<var>form</var>/</code>
  250.  250 
  251.  251 <p><dfn>If you submit the form you should simply see the form reload.  That's because you haven't set up any validation
  252.  252 rules yet.</dfn></p>
  253.  253 
  254.  254 <p><strong>Since you haven't told the Form Validation class to validate anything yet, it returns <kbd>FALSE</kbd> (boolean false) by default.  The <samp>run()</samp>
  255.  255 function only returns <kbd>TRUE</kbd> if it has successfully applied your rules without any of them failing.</strong></p>
  256.  256 
  257.  257 
  258.  258 <h2>Explanation</h2>
  259.  259 
  260.  260 <p>You'll notice several things about the above pages:</p>
  261.  261 
  262.  262 <p>The <dfn>form</dfn> (myform.php) is a standard web form with a couple exceptions:</p>
  263.  263 
  264.  264 <ol>
  265.  265 <li>It uses a <dfn>form helper</dfn> to create the form opening.
  266.  266 Technically, this isn't necessary.  You could create the form using standard HTML.  However, the benefit of using the helper
  267.  267 is that it generates the action URL for you, based on the URL in your config file.  This makes your application more portable in the event your URLs change.</li>
  268.  268 
  269.  269 <li>At the top of the form you'll notice the following function call:
  270.  270 <code>&lt;?php echo validation_errors(); ?&gt;</code>
  271.  271 
  272.  272 <p>This function will return any error messages sent back by the validator. If there are no messages it returns an empty string.</p>
  273.  273 </li>
  274.  274 </ol>
  275.  275 
  276.  276 <p>The <dfn>controller</dfn> (form.php) has one function: <dfn>index()</dfn>. This function initializes the validation class and
  277.  277 loads the <var>form helper</var> and <var>URL helper</var> used by your view files. It also <samp>runs</samp>
  278.  278 the validation routine. Based on
  279.  279 whether the validation was successful it either presents the form or the success page.</p>
  280.  280 
  281.  281 
  282.  282 
  283.  283 
  284.  284 <a name="validationrules"></a>
  285.  285 
  286.  286 <h2>Setting Validation Rules</h2>
  287.  287 
  288.  288 <p>CodeIgniter lets you set as many validation rules as you need for a given field, cascading them in order, and it even lets you prep and pre-process the field data
  289.  289 at the same time. To set validation rules you will use the <dfn>set_rules()</dfn> function:</p>
  290.  290 
  291.  291 <code>$this->form_validation->set_rules();</code>
  292.  292 
  293.  293 <p>The above function takes <strong>three</strong> parameters as input:</p>
  294.  294 
  295.  295 <ol>
  296.  296  <li>The field name - the exact name you've given the form field.</li>
  297.  297  <li>A "human" name for this field, which will be inserted into the error message. For example, if your field is named "user" you might give it a human name of "Username". <strong>Note:</strong> If you would like the field name to be stored in a language file, please see <a href="#translatingfn">Translating Field Names</a>.</li>
  298.  298  <li>The validation rules for this form field.</li>
  299.  299 </ol>
  300.  300 
  301.  301 
  302.  302 <p><br />Here is an example. In your <dfn>controller</dfn> (form.php), add this code just below the validation initialization function:</p>
  303.  303 
  304.  304 <code>
  305.  305 $this->form_validation->set_rules('username', 'Username', 'required');<br />
  306.  306 $this->form_validation->set_rules('password', 'Password', 'required');<br />
  307.  307 $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');<br />
  308.  308 $this->form_validation->set_rules('email', 'Email', 'required');<br />
  309.  309 </code>
  310.  310 
  311.  311 <p>Your controller should now look like this:</p>
  312.  312 
  313.  313 <textarea class="textarea" style="width:100%" cols="50" rows="28">&lt;?php
  314.  314 
  315.  315 class Form extends CI_Controller {
  316.  316 
  317.  317  function index()
  318.  318  {
  319.  319  $this->load->helper(array('form', 'url'));
  320.  320 
  321.  321  $this->load->library('form_validation');
  322.  322 
  323.  323  $this->form_validation->set_rules('username', 'Username', 'required');
  324.  324  $this->form_validation->set_rules('password', 'Password', 'required');
  325.  325  $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
  326.  326  $this->form_validation->set_rules('email', 'Email', 'required');
  327.  327 
  328.  328  if ($this->form_validation->run() == FALSE)
  329.  329  {
  330.  330  $this->load->view('myform');
  331.  331  }
  332.  332  else
  333.  333  {
  334.  334  $this->load->view('formsuccess');
  335.  335  }
  336.  336  }
  337.  337 }
  338.  338 ?></textarea>
  339.  339 
  340.  340 <p><dfn>Now submit the form with the fields blank and you should see the error messages.
  341.  341 If you submit the form with all the fields populated you'll see your success page.</dfn></p>
  342.  342 
  343.  343 <p class="important"><strong>Note:</strong> The form fields are not yet being re-populated with the data when
  344.  344 there is an error.  We'll get to that shortly.</p>

  349.  349 <a name="validationrulesasarray"></a>
  350.  350 <h2>Setting Rules Using an Array</h2>
  351.  351 
  352.  352 <p>Before moving on it should be noted that the rule setting function can be passed an array if you prefer to set all your rules in one action.
  353.  353 If you use this approach you must name your array keys as indicated:</p>
  354.  354 
  355.  355 <code>
  356.  356 $config = array(<br />
  357.  357 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array(<br />

  414.  414 
  415.  415 <p>In addition to the validation functions like the ones we used above, you can also prep your data in various ways.
  416.  416 For example, you can set up rules like this:</p>
  417.  417 
  418.  418 <code>
  419.  419 $this->form_validation->set_rules('username', 'Username', '<kbd>trim</kbd>|required|min_length[5]|max_length[12]|<kbd>xss_clean</kbd>');<br />

  433.  433 the validation rules so if there is an error, the original data will be shown in the form.</p>

  502.  502 
  503.  503 <p>The validation system supports callbacks to your own validation functions.  This permits you to extend the validation class
  504.  504 to meet your needs.  For example, if you need to run a database query to see if the user is choosing a unique username, you can
  505.  505 create a callback function that does that.  Let's create a example of this.</p>
  506.  506 
  507.  507 <p>In your controller, change the "username" rule to this:</p>

  510.  510 
  511.  511 <p>Then add a new function called <dfn>username_check</dfn> to your controller.  Here's how your controller should now look:</p>
  512.  512 
  513.  513 <textarea class="textarea" style="width:100%" cols="50" rows="40">&lt;?php
  514.  514

  566.  566

  581.  581 
  582.  582 <p>You can also override any error message found in the language file.  For example, to change the message for the "required" rule you will do this:</p>

  858.  858 <h1>Using Arrays as Field Names</h1>
  859.  859 
  860.  860 <p>The Form Validation class supports the use of arrays as field names.  Consider this example:</p>
  861.  861 
  862.  862 <code>&lt;input type="text" name="<kbd>options[]</kbd>" value="" size="50" /></code>
  863.  863 
  864.  864 <p>If you do use an array as a field name, you must use the EXACT array name in the <a href="#helperreference">Helper Functions</a> that require the field name,
  865.  865 and as your Validation Rule field name.</p>
  866.  866 
  867.  867 <p>For example, to set a rule for the above field you would use:</p>
  868.  868 
  869.  869 <code>$this->form_validation->set_rules('<kbd>options[]</kbd>', 'Options', 'required');</code>
  870.  870 
  871.  871 <p>Or, to show an error for the above field you would use:</p>
  872.  872 
  873.  873 <code>&lt;?php echo form_error('<kbd>options[]</kbd>'); ?></code>
  874.  874 
  875.  875 <p>Or to re-populate the field you would use:</p>
  876.  876 
  877.  877 <code>&lt;input type="text" name="<kbd>options[]</kbd>" value="<kbd>&lt;?php echo set_value('<kbd>options[]</kbd>'); ?></kbd>" size="50" /></code>
  878.  878 
  879.  879 <p>You can use multidimensional arrays as field names as well. For example:</p>
  880.  880 
  881.  881 <code>&lt;input type="text" name="<kbd>options[size]</kbd>" value="" size="50" /></code>
  882.  882 
  883.  883 <p>Or even:</p>
  884.  884 
  885.  885 <code>&lt;input type="text" name="<kbd>sports[nba][basketball]</kbd>" value="" size="50" /></code>
  886.  886 
  887.  887 <p>As with our first example, you must use the exact array name in the helper functions:</p>
  888.  888 
  889.  889 <code>&lt;?php echo form_error('<kbd>sports[nba][basketball]</kbd>'); ?></code>
  890.  890 
  891.  891 <p>If you are using checkboxes (or other fields) that have multiple options, don't forget to leave an empty bracket after each option, so that all selections will be added to the
  892.  892 POST array:</p>
  893.  893 
  894.  894 <code>
  895.  895 &lt;input type="checkbox" name="<kbd>options[]</kbd>" value="red" /><br />
  896.  896 &lt;input type="checkbox" name="<kbd>options[]</kbd>" value="blue" /><br />
  897.  897 &lt;input type="checkbox" name="<kbd>options[]</kbd>" value="green" />
  898.  898 </code>
  899.  899 
  900.  900 <p>Or if you use a multidimensional array:</p>
  901.  901 
  902.  902 <code>
  903.  903 &lt;input type="checkbox" name="<kbd>options[color][]</kbd>" value="red" /><br />
  904.  904 &lt;input type="checkbox" name="<kbd>options[color][]</kbd>" value="blue" /><br />
  905.  905 &lt;input type="checkbox" name="<kbd>options[color][]</kbd>" value="green" />
  906.  906 </code>
  907.  907 
  908.  908 <p>When you use a helper function you'll include the bracket as well:</p>
  909.  909 
  910.  910 <code>&lt;?php echo form_error('<kbd>options[color][]</kbd>'); ?></code>
