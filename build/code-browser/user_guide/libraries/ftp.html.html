  59.  59 <h1>FTP Class</h1>
  60.  60 
  61.  61 <p>CodeIgniter's FTP Class permits files to be transfered to a remote server. Remote files can also be moved, renamed,
  62.  62 and deleted.  The FTP class also includes a "mirroring" function that permits an entire local directory to be recreated remotely via FTP.</p>
  63.  63 
  64.  64 <p class="important"><strong>Note:</strong>&nbsp; SFTP and SSL FTP protocols are not supported, only standard FTP.</p>
  65.  65 
  66.  66 <h2>Initializing the Class</h2>
  67.  67 
  68.  68 <p>Like most other classes in CodeIgniter, the FTP class is initialized in your controller using the <dfn>$this->load->library</dfn> function:</p>
  69.  69 
  70.  70 <code>$this->load->library('ftp');</code>
  71.  71 <p>Once loaded, the FTP object will be available using: <dfn>$this->ftp</dfn></p>
  72.  72 
  73.  73 
  74.  74 <h2>Usage Examples</h2>
  75.  75 
  76.  76 <p>In this example a connection is opened to the FTP server, and a local file is read and uploaded in ASCII mode. The
  77.  77 file permissions are set to 755.</p>
  78.  78 
  79.  79 <code>
  80.  80 $this->load->library('ftp');<br />
  81.  81 <br />
  82.  82 $config['hostname'] = 'ftp.example.com';<br />
  83.  83 $config['username'] = 'your-username';<br />
  84.  84 $config['password'] = 'your-password';<br />
  85.  85 $config['debug'] = TRUE;<br />
  86.  86 <br />
  87.  87 $this->ftp->connect($config);<br />
  88.  88 <br />
  89.  89 $this->ftp->upload('/local/path/to/myfile.html', '/public_html/myfile.html', 'ascii', 0775);<br />
  90.  90 <br />
  91.  91 $this->ftp->close();
  92.  92 
  93.  93 </code>
  94.  94 
  95.  95 
  96.  96 <p>In this example a list of files is retrieved from the server.</p>
  97.  97 
  98.  98 <code>
  99.  99 $this->load->library('ftp');<br />
  100. 100 <br />
  101. 101 $config['hostname'] = 'ftp.example.com';<br />
  102. 102 $config['username'] = 'your-username';<br />
  103. 103 $config['password'] = 'your-password';<br />
  104. 104 $config['debug'] = TRUE;<br />
  105. 105 <br />
  106. 106 $this->ftp->connect($config);<br />
  107. 107 <br />
  108. 108 $list = $this->ftp->list_files('/public_html/');<br />
  109. 109 <br />
  110. 110 print_r($list);<br />
  111. 111 <br />
  112. 112 $this->ftp->close();
  113. 113 </code>
  114. 114 
  115. 115 <p>In this example a local directory is mirrored on the server.</p>
  116. 116 
  117. 117 
  118. 118 <code>
  119. 119 $this->load->library('ftp');<br />
  120. 120 <br />
  121. 121 $config['hostname'] = 'ftp.example.com';<br />
  122. 122 $config['username'] = 'your-username';<br />
  123. 123 $config['password'] = 'your-password';<br />
  124. 124 $config['debug'] = TRUE;<br />
  125. 125 <br />
  126. 126 $this->ftp->connect($config);<br />
  127. 127 <br />
  128. 128 $this->ftp->mirror('/path/to/myfolder/', '/public_html/myfolder/');<br />
  129. 129 <br />
  130. 130 $this->ftp->close();
  131. 131 </code>
  132. 132 
  133. 133 
  134. 134 <h1>Function Reference</h1>
  135. 135 
  136. 136 <h2>$this->ftp->connect()</h2>
  137. 137 
  138. 138 <p>Connects and logs into to the FTP server. Connection preferences are set by passing an array
  139. 139 to the function, or you can store them in a config file.</p>
  140. 140 
  141. 141 
  142. 142 <p>Here is an example showing how you set preferences manually:</p>
  143. 143 
  144. 144 <code>
  145. 145 $this->load->library('ftp');<br />
  146. 146 <br />
  147. 147 $config['hostname'] = 'ftp.example.com';<br />
  148. 148 $config['username'] = 'your-username';<br />
  149. 149 $config['password'] = 'your-password';<br />
  150. 150 $config['port']&nbsp;&nbsp;&nbsp;&nbsp; = 21;<br />
  151. 151 $config['passive']&nbsp;&nbsp;= FALSE;<br />
  152. 152 $config['debug']&nbsp;&nbsp;&nbsp;&nbsp;= TRUE;<br />
  153. 153 <br />
  154. 154 $this->ftp->connect($config);<br />
  155. 155 </code>
  156. 156 
  157. 157 <h3>Setting FTP Preferences in a Config File</h3>
  158. 158 
  159. 159 <p>If you prefer you can store your FTP preferences in a config file.
  160. 160 Simply create a new file called the <var>ftp.php</var>,  add the <var>$config</var>
  161. 161 array in that file. Then save the file at <var>config/ftp.php</var> and it will be used automatically.</p>
  162. 162 
  163. 163 <h3>Available connection options:</h3>
  164. 164 
  165. 165 
  166. 166 <ul>
  167. 167 <li><strong>hostname</strong> - the FTP hostname.  Usually something like:&nbsp; <dfn>ftp.example.com</dfn></li>
  168. 168 <li><strong>username</strong> - the FTP username.</li>
  169. 169 <li><strong>password</strong> - the FTP password.</li>
  170. 170 <li><strong>port</strong> - The port number. Set to <dfn>21</dfn> by default.</li>
  171. 171 <li><strong>debug</strong> - <kbd>TRUE/FALSE</kbd> (boolean). Whether to enable debugging to display error messages.</li>
  172. 172 <li><strong>passive</strong> - <kbd>TRUE/FALSE</kbd> (boolean). Whether to use passive mode.  Passive is set automatically by default.</li>
  173. 173 </ul>
  174. 174 
  175. 175 
  176. 176 
  177. 177 <h2>$this->ftp->upload()</h2>
  178. 178 
  179. 179 <p>Uploads a file to your server.  You must supply the local path and the remote path, and you can optionally set the mode and permissions.
  180. 180 Example:</p>
  181. 181 
  182. 182 
  183. 183 <code>$this->ftp->upload('/local/path/to/myfile.html', '/public_html/myfile.html', 'ascii', 0775);</code>
  184. 184 
  185. 185 <p><strong>Mode options are:</strong>&nbsp; <kbd>ascii</kbd>, <kbd>binary</kbd>, and <kbd>auto</kbd> (the default). If
  186. 186 <kbd>auto</kbd> is used it will base the mode on the file extension of the source file.</p>
  187. 187 
  188. 188 <p>Permissions can be passed as an <kbd>octal</kbd> value in the fourth parameter.</p>
  189. 189 
  190. 190 <h2>$this->ftp->download()</h2>
  191. 191 
  192. 192 <p>Downloads a file from your server.  You must supply the remote path and the local path, and you can optionally set the mode.
  193. 193 Example:</p>
  194. 194 
  195. 195 <code>$this->ftp->download('/public_html/myfile.html', '/local/path/to/myfile.html', 'ascii');</code>
  196. 196 
  197. 197 <p><strong>Mode options are:</strong>&nbsp; <kbd>ascii</kbd>, <kbd>binary</kbd>, and <kbd>auto</kbd> (the default). If
  198. 198 <kbd>auto</kbd> is used it will base the mode on the file extension of the source file.</p>
  199. 199 
  200. 200 <p>Returns FALSE if the download does not execute successfully (including if PHP does not have permission to write the local file)</p>
  201. 201 
  202. 202 
  203. 203 <h2>$this->ftp->rename()</h2>
  204. 204 <p>Permits you to rename a file.  Supply the source file name/path and the new file name/path.</p>
  205. 205 
  206. 206 <code>
  207. 207 // Renames green.html to blue.html<br />
  208. 208 $this->ftp->rename('/public_html/foo/green.html', '/public_html/foo/blue.html');
  209. 209 </code>
  210. 210 
  211. 211 <h2>$this->ftp->move()</h2>
  212. 212 <p>Lets you move a file.  Supply the source and destination paths:</p>
  213. 213 
  214. 214 <code>
  215. 215 // Moves blog.html from "joe" to "fred"<br />
  216. 216 $this->ftp->move('/public_html/joe/blog.html', '/public_html/fred/blog.html');
  217. 217 </code>
  218. 218 
  219. 219 <p>Note: if the destination file name is different the file will be renamed.</p>
  220. 220 
  221. 221 
  222. 222 <h2>$this->ftp->delete_file()</h2>
  223. 223 <p>Lets you delete a file.  Supply the source path with the file name.</p>
  224. 224 
  225. 225 <code>
  226. 226 $this->ftp->delete_file('/public_html/joe/blog.html');
  227. 227 </code>
  228. 228 
  229. 229 
  230. 230 <h2>$this->ftp->delete_dir()</h2>
  231. 231 <p>Lets you delete a directory and everything it contains.  Supply the source path to the directory with a trailing slash.</p>
  232. 232 
  233. 233 <p class="important"><strong>Important</strong>&nbsp; Be VERY careful with this function.  It will recursively delete
  234. 234 <b>everything</b> within the supplied path, including sub-folders and all files.  Make absolutely sure your path is correct.
  235. 235 Try using the <kbd>list_files()</kbd> function first to verify that your path is correct.</p>
  236. 236 
  237. 237 <code>
  238. 238 $this->ftp->delete_dir('/public_html/path/to/folder/');
  239. 239 </code>
  240. 240 
  241. 241 
  242. 242 
  243. 243 <h2>$this->ftp->list_files()</h2>
  244. 244 <p>Permits you to retrieve a list of files on your server returned as an <dfn>array</dfn>.  You must supply
  245. 245 the path to the desired directory.</p>
  246. 246 
  247. 247 <code>
  248. 248 $list = $this->ftp->list_files('/public_html/');<br />
  249. 249 <br />
  250. 250 print_r($list);
  251. 251 </code>
  252. 252 
  253. 253 
  254. 254 <h2>$this->ftp->mirror()</h2>
  255. 255 
  256. 256 <p>Recursively reads a local folder and everything it contains (including sub-folders) and creates a
  257. 257 mirror via FTP based on it.  Whatever the directory structure of the original file path will be recreated on the server.
  258. 258 You must supply a source path and a destination path:</p>
  259. 259 
  260. 260 <code>
  261. 261 $this->ftp->mirror('/path/to/myfolder/', '/public_html/myfolder/');
  262. 262 </code>
  263. 263 
  264. 264 
  265. 265 
  266. 266 <h2>$this->ftp->mkdir()</h2>
  267. 267 
  268. 268 <p>Lets you create a directory on your server.  Supply the path ending in the folder name you wish to create, with a trailing slash.
  269. 269 Permissions can be set by passed an <kbd>octal</kbd> value in the second parameter.</p>
  270. 270 
  271. 271 <code>
  272. 272 // Creates a folder named "bar"<br />
  273. 273 $this->ftp->mkdir('/public_html/foo/bar/', DIR_WRITE_MODE);
  274. 274 </code>
  275. 275 
  276. 276 
  277. 277 <h2>$this->ftp->chmod()</h2>
  278. 278 
  279. 279 <p>Permits you to set file permissions.  Supply the path to the file or folder you wish to alter permissions on:</p>
  280. 280 
  281. 281 <code>
  282. 282 // Chmod "bar" to 777<br />
  283. 283 $this->ftp->chmod('/public_html/foo/bar/', DIR_WRITE_MODE);
  284. 284 </code>
  285. 285 
  286. 286 
  287. 287 
  288. 288 
  289. 289 <h2>$this->ftp->close();</h2>
  290. 290 <p>Closes the connection to your server.  It's recommended that you use this when you are finished uploading.</p>
