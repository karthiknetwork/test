  59.  59 <h1>Image Manipulation Class</h1>
  60.  60 
  61.  61 <p>CodeIgniter's Image Manipulation class lets you perform the following actions:</p>
  62.  62 
  63.  63 <ul>
  64.  64 <li>Image Resizing</li>
  65.  65 <li>Thumbnail Creation</li>
  66.  66 <li>Image Cropping</li>
  67.  67 <li>Image Rotating</li>
  68.  68 <li>Image Watermarking</li>
  69.  69 </ul>
  70.  70 
  71.  71 <p>All three major image libraries are supported:  GD/GD2, NetPBM, and ImageMagick</p>
  72.  72 
  73.  73 <p class="important"><strong>Note:</strong> Watermarking is only available using the GD/GD2 library.
  74.  74 In addition, even though other libraries are supported, GD is required in
  75.  75 order for the script to calculate the image properties.  The image processing, however, will be performed with the
  76.  76 library you specify.</p>
  77.  77 
  78.  78 
  79.  79 <h2>Initializing the Class</h2>
  80.  80 
  81.  81 <p>Like most other classes in CodeIgniter, the image class is initialized in your controller
  82.  82 using the <dfn>$this->load-&gt;library</dfn> function:</p>
  83.  83 <code>$this->load->library('image_lib');</code>
  84.  84 
  85.  85 <p>Once the library is loaded it will be ready for use.  The image library object you will use to call all functions is: <dfn>$this->image_lib</dfn></p>
  86.  86 
  87.  87 
  88.  88 <h2>Processing an Image</h2>
  89.  89 
  90.  90 <p>Regardless of the type of processing you would like to perform (resizing, cropping, rotation, or watermarking), the general process is
  91.  91 identical. You will set some preferences corresponding to the action you intend to perform, then
  92.  92 call one of four available processing functions.  For example, to create an image thumbnail you'll do this:</p>
  93.  93 
  94.  94 <code>$config['image_library'] = 'gd2';<br />
  95.  95 $config['source_image'] = '/path/to/image/mypic.jpg';<br />
  96.  96 $config['create_thumb'] = TRUE;<br />
  97.  97 $config['maintain_ratio'] = TRUE;<br />
  98.  98 $config['width'] = 75;<br />
  99.  99 $config['height'] = 50;<br />
  100. 100 <br />
  101. 101 $this->load->library('image_lib', $config);
  102. 102 <br />
  103. 103 <br />
  104. 104 $this->image_lib->resize();</code>
  105. 105 
  106. 106 <p>The above code tells the <dfn>image_resize</dfn> function to look for an image called <em>mypic.jpg</em>
  107. 107 located in the <dfn>source_image</dfn> folder, then create a thumbnail that is 75 X 50 pixels using the GD2 <dfn>image_library</dfn>.
  108. 108 Since the <dfn>maintain_ratio</dfn> option is enabled, the thumb will be as close to the target <dfn>width</dfn> and
  109. 109 <dfn>height</dfn> as possible while preserving the original aspect ratio.  The thumbnail will be called <em>mypic_thumb.jpg</em>
  110. 110 </p>
  111. 111 
  112. 112 <p class="important"><strong>Note:</strong> In order for the image class to be allowed to do any processing, the
  113. 113 folder containing the image files must have write permissions.</p>
  114. 114 
  115. 115 <p class="important"><strong>Note:</strong> Image processing can require a considerable amount of server memory for some operations. If you are experiencing out of memory errors while processing images you may need to limit their maximum size, and/or adjust PHP memory limits.</p>
  116. 116 
  117. 117 <h2>Processing Functions</h2>
  118. 118 
  119. 119 <p>There are four available processing functions:</p>
  120. 120 
  121. 121 <ul>
  122. 122 <li>$this->image_lib->resize()</li>
  123. 123 <li>$this->image_lib->crop()</li>
  124. 124 <li>$this->image_lib->rotate()</li>
  125. 125 <li>$this->image_lib->watermark()</li>
  126. 126 <li>$this-&gt;image_lib-&gt;clear()</li>
  127. 127 </ul>
  128. 128 
  129. 129 <p>These functions return boolean TRUE upon success and FALSE for failure.  If they fail you can retrieve the
  130. 130 error message using this function:</p>
  131. 131 
  132. 132 <code>echo $this->image_lib->display_errors();</code>
  133. 133 
  134. 134 <p>A good practice is use the processing function conditionally, showing an error upon failure, like this:</p>
  135. 135 
  136. 136 <code>if ( ! $this->image_lib->resize())<br />
  137. 137 {<br />
  138. 138 &nbsp;&nbsp;&nbsp;&nbsp;echo $this->image_lib->display_errors();<br />
  139. 139 }</code>
  140. 140 
  141. 141 <p>Note:  You can optionally specify the HTML formatting to be applied to the errors, by submitting the opening/closing
  142. 142 tags in the function, like this:</p>
  143. 143 
  144. 144 <code>$this->image_lib->display_errors('<var>&lt;p></var>', '<var>&lt;/p></var>');</code>
  145. 145 
  146. 146 
  147. 147 <h2>Preferences</h2>
  148. 148 
  149. 149 <p>The  preferences described below allow you to tailor the image processing to suit your needs.</p>
  150. 150 
  151. 151 <p>Note that not all preferences are available for every
  152. 152 function.  For example, the x/y axis preferences are only available for image cropping. Likewise, the width and height
  153. 153 preferences have no effect on cropping.  The "availability" column indicates which functions support a given preference.</p>
  154. 154 
  155. 155 <p>Availability Legend:</p>
  156. 156 
  157. 157 <ul>
  158. 158 <li><var>R</var> - Image Resizing</li>
  159. 159 <li><var>C</var> - Image Cropping</li>
  160. 160 <li><var>X</var> - Image Rotation</li>
  161. 161 <li><var>W</var> - Image Watermarking</li>
  162. 162 
  163. 163 </ul>

  305. 305 
  306. 306 <h2>Setting preferences in a config file</h2>
  307. 307 
  308. 308 <p>If you prefer not to set preferences using the above method, you can instead put them into a config file.
  309. 309 Simply create a new file called <var>image_lib.php</var>,  add the <var>$config</var>
  310. 310 array in that file. Then save the file in: <var>config/image_lib.php</var> and it will be used automatically. You
  311. 311 will NOT need to use the <dfn>$this->image_lib->initialize</dfn> function if you save your preferences in a config file.</p>
  312. 312 
  313. 313 
  314. 314 <h2>$this->image_lib->resize()</h2>
  315. 315 
  316. 316 <p>The image resizing function lets you resize the original image, create a copy (with or without resizing),
  317. 317 or create a thumbnail image.</p>
  318. 318 
  319. 319 <p>For practical purposes there is no difference between creating a copy and creating
  320. 320 a thumbnail except a thumb will have the thumbnail marker as part of the name (ie, mypic_thumb.jpg).</p>
  321. 321 
  322. 322 <p>All preferences listed in the table above are available for this function except these three:  rotation_angle, x_axis, and y_axis.</p>
  323. 323 
  324. 324 <h3>Creating a Thumbnail</h3>
  325. 325 
  326. 326 <p>The resizing function will create a thumbnail file (and preserve the original) if you set this preference to TRUE:</p>
  327. 327 
  328. 328 <code>$config['create_thumb'] = TRUE;</code>
  329. 329 
  330. 330 <p>This single preference determines whether a thumbnail is created or not.</p>
  331. 331 
  332. 332 <h3>Creating a Copy</h3>
  333. 333 
  334. 334 <p>The resizing function will create a copy of the image file (and preserve the original) if you set
  335. 335 a path and/or a new filename using this preference:</p>
  336. 336 
  337. 337 <code>$config['new_image'] = '/path/to/new_image.jpg';</code>
  338. 338 
  339. 339 <p>Notes regarding this preference:</p>
  340. 340 <ul>
  341. 341 <li>If only the new image name is specified it will be placed in the same folder as the original</li>
  342. 342 <li>If only the path is specified, the new image will be placed in the destination with the same name as the original.</li>
  343. 343 <li>If both the path and image name are specified it will placed in its own destination and given the new name.</li>
  344. 344 </ul>
  345. 345 
  346. 346 
  347. 347 <h3>Resizing the Original Image</h3>
  348. 348 
  349. 349 <p>If neither of the two preferences listed above (create_thumb, and new_image) are used, the resizing function will instead
  350. 350 target the original image for processing.</p>
  351. 351 
  352. 352 
  353. 353 <h2>$this->image_lib->crop()</h2>
  354. 354 
  355. 355 <p>The cropping function works nearly identically to the resizing function except it requires that you set
  356. 356 preferences for the X and Y axis (in pixels) specifying where to crop, like this:</p>
  357. 357 
  358. 358 <code>$config['x_axis'] = '100';<br />
  359. 359 $config['y_axis'] = '40';</code>
  360. 360 
  361. 361 <p>All preferences listed in the table above are available for this function except these:  rotation_angle, width, height, create_thumb, new_image.</p>
  362. 362 
  363. 363 <p>Here's an example showing how you might crop an image:</p>
  364. 364 
  365. 365 <code>$config['image_library'] = 'imagemagick';<br />
  366. 366 $config['library_path'] = '/usr/X11R6/bin/';<br />
  367. 367 $config['source_image'] = '/path/to/image/mypic.jpg';<br />
  368. 368 $config['x_axis'] = '100';<br />
  369. 369 $config['y_axis'] = '60';<br />
  370. 370 <br />
  371. 371 $this->image_lib->initialize($config);
  372. 372 <br />
  373. 373 <br />
  374. 374 if ( ! $this->image_lib->crop())<br />
  375. 375 {<br />
  376. 376 &nbsp;&nbsp;&nbsp;&nbsp;echo $this->image_lib->display_errors();<br />
  377. 377 }</code>
  378. 378 
  379. 379 
  380. 380 <p>Note: Without a visual interface it is difficult to crop images, so this function is not very useful
  381. 381 unless you intend to build such an interface.  That's exactly what we did using for the photo
  382. 382 gallery module in ExpressionEngine, the CMS we develop.  We added a JavaScript UI that lets the cropping
  383. 383 area be selected.</p>
  384. 384 
  385. 385 <h2>$this->image_lib->rotate()</h2>
  386. 386 
  387. 387 <p>The image rotation function requires that the angle of rotation be set via its preference:</p>
  388. 388 
  389. 389 <code>$config['rotation_angle'] = '90';</code>
  390. 390 
  391. 391 <p>There are 5 rotation options:</p>
  392. 392 
  393. 393 <ol>
  394. 394 <li>90 - rotates counter-clockwise by 90 degrees.</li>
  395. 395 <li>180 - rotates counter-clockwise by 180 degrees.</li>
  396. 396 <li>270 - rotates counter-clockwise by 270 degrees.</li>
  397. 397 <li>hor - flips the image horizontally.</li>
  398. 398 <li>vrt - flips the image vertically.</li>
  399. 399 </ol>
  400. 400 
  401. 401 <p>Here's an example showing how you might rotate an image:</p>
  402. 402 
  403. 403 <code>$config['image_library'] = 'netpbm';<br />
  404. 404 $config['library_path'] = '/usr/bin/';<br />
  405. 405 $config['source_image'] = '/path/to/image/mypic.jpg';<br />
  406. 406 $config['rotation_angle'] = 'hor';<br />
  407. 407 <br />
  408. 408 $this->image_lib->initialize($config);
  409. 409 <br />
  410. 410 <br />
  411. 411 if ( ! $this->image_lib->rotate())<br />
  412. 412 {<br />
  413. 413 &nbsp;&nbsp;&nbsp;&nbsp;echo $this->image_lib->display_errors();<br />
  414. 414 }</code>
  415. 415 
  416. 416 
  417. 417 
  418. 418 <h2>$this-&gt;image_lib-&gt;clear()</h2>
  419. 419 <p>The clear function resets all of the values used when processing an image. You will want to call this if you are processing images in a loop.</p>
  420. 420 <p><code>$this-&gt;image_lib-&gt;clear();</code></p>

  422. 422 <h1>Image Watermarking</h1>
  423. 423 
  424. 424 <p>The Watermarking feature requires the GD/GD2 library.</p>
  425. 425 
  426. 426 
  427. 427 <h2>Two Types of Watermarking</h2>
  428. 428 
  429. 429 <p>There are two types of watermarking that you can use:</p>
  430. 430 
  431. 431 <ul>
  432. 432 <li><strong>Text</strong>: The watermark message will be generating using text, either with a True Type font that you specify, or
  433. 433 using the native text output that the GD library supports. If you use the True Type version your GD installation
  434. 434 must be compiled with True Type support (most are, but not all).</li>
  435. 435 
  436. 436 <li><strong>Overlay</strong>: The watermark message will be generated by overlaying an image (usually a transparent PNG or GIF)
  437. 437 containing your watermark over the source image.</li>
  438. 438 
  439. 439 </ul>
  440. 440 
  441. 441 
  442. 442 <h2>Watermarking an Image</h2>
  443. 443 
  444. 444 <p>Just as with the other functions (resizing, cropping, and rotating) the general process for watermarking
  445. 445 involves setting the preferences corresponding to the action you intend to perform, then
  446. 446 calling the watermark function.  Here is an example:</p>
  447. 447 
  448. 448 <code>
  449. 449 $config['source_image'] = '/path/to/image/mypic.jpg';<br />
  450. 450 $config['wm_text'] = 'Copyright 2006 - John Doe';<br />
  451. 451 $config['wm_type'] = 'text';<br />
  452. 452 $config['wm_font_path'] = './system/fonts/texb.ttf';<br />
  453. 453 $config['wm_font_size'] = '16';<br />
  454. 454 $config['wm_font_color'] = 'ffffff';<br />
  455. 455 $config['wm_vrt_alignment']  = 'bottom';<br />
  456. 456 $config['wm_hor_alignment']  = 'center';<br />
  457. 457 $config['wm_padding']  = '20';<br />
  458. 458 <br />
  459. 459 $this->image_lib->initialize($config);
  460. 460 <br />
  461. 461 <br />
  462. 462 $this->image_lib->watermark();</code>
  463. 463 
  464. 464 
  465. 465 <p>The above example will use a 16 pixel True Type font to create the text "Copyright 2006 - John Doe".  The watermark
  466. 466 will be positioned at the bottom/center of the image, 20 pixels from the bottom of the image.</p>
  467. 467 
  468. 468 <p class="important"><strong>Note:</strong> In order for the image class to be allowed to do any processing, the image file must have &quot;write&quot; file permissions. For example, 777.</p>
