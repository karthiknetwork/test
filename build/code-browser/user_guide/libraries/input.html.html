  59.  59 <h1>Input Class</h1>
  60.  60 
  61.  61 <p>The Input Class serves two purposes:</p>
  62.  62 
  63.  63 <ol>
  64.  64 <li>It pre-processes global input data for security.</li>
  65.  65 <li>It provides some helper functions for fetching input data and pre-processing it.</li>
  66.  66 </ol>
  67.  67 
  68.  68 <p class="important"><strong>Note:</strong> This class is initialized automatically by the system so there is no need to do it manually.</p>
  69.  69 
  70.  70 
  71.  71 <h2>Security Filtering</h2>
  72.  72 
  73.  73 <p>The security filtering function is called automatically when a new <a href="../general/controllers.html">controller</a> is invoked.  It does the following:</p>
  74.  74 
  75.  75 <ul>
  76.  76 <li>If $config['allow_get_array'] is FALSE(default is TRUE), destroys the global GET array.</li>
  77.  77 <li>Destroys all global variables in the event register_globals is turned on.</li>
  78.  78 <li>Filters the GET/POST/COOKIE array keys, permitting only alpha-numeric (and a few other) characters.</li>
  79.  79 <li>Provides XSS (Cross-site Scripting Hacks) filtering.  This can be enabled globally, or upon request.</li>
  80.  80 <li>Standardizes newline characters to \n(In Windows \r\n)</li>
  81.  81 </ul>
  82.  82 
  83.  83 
  84.  84 <h2>XSS Filtering</h2>
  85.  85 
  86.  86 <p>The Input class has the ability to filter input automatically to prevent cross-site scripting attacks.  If you want the filter to run automatically every time it encounters POST or COOKIE data you can enable it by opening your
  87.  87 <kbd>application/config/config.php</kbd> file and setting this:</p>
  88.  88 
  89.  89 <code>$config['global_xss_filtering'] = TRUE;</code>
  90.  90 
  91.  91 <p>Please refer to the <a href="security.html">Security class</a> documentation for information on using XSS Filtering in your application.</p>
  92.  92 
  93.  93 
  94.  94 <h2>Using POST, COOKIE, or SERVER Data</h2>
  95.  95 
  96.  96 <p>CodeIgniter comes with three helper functions that let you fetch POST, COOKIE or SERVER items.  The main advantage of using the provided
  97.  97 functions rather than fetching an item directly ($_POST['something']) is that the functions will check to see if the item is set and
  98.  98 return false (boolean) if not.  This lets you conveniently use data without having to test whether an item exists first.
  99.  99 In other words, normally you might do something like this:</p>
  100. 100 
  101. 101 <code>
  102. 102 if ( ! isset($_POST['something']))<br />
  103. 103 {<br />
  104. 104 &nbsp;&nbsp;&nbsp;&nbsp;$something = FALSE;<br />
  105. 105 }<br />
  106. 106 else<br />
  107. 107 {<br />
  108. 108 &nbsp;&nbsp;&nbsp;&nbsp;$something = $_POST['something'];<br />
  109. 109 }</code>
  110. 110 
  111. 111 <p>With CodeIgniter's built in functions you can simply do this:</p>
  112. 112 
  113. 113 <code>$something = $this->input->post('something');</code>
  114. 114 
  115. 115 <p>The three functions are:</p>
  116. 116 
  117. 117 <ul>
  118. 118 <li>$this->input->post()</li>
  119. 119 <li>$this->input->cookie()</li>
  120. 120 <li>$this->input->server()</li>
  121. 121 </ul>
  122. 122 
  123. 123 <h2>$this->input->post()</h2>
  124. 124 
  125. 125 <p>The first parameter will contain the name of the POST item you are looking for:</p>
  126. 126 
  127. 127 <code>$this->input->post('some_data');</code>
  128. 128 
  129. 129 <p>The function returns FALSE (boolean) if the item you are attempting to retrieve does not exist.</p>
  130. 130 
  131. 131 <p>The second optional parameter lets you run the data through the XSS filter.  It's enabled by setting the second parameter to boolean TRUE;</p>
  132. 132 
  133. 133 <code>$this->input->post('some_data', TRUE);</code>
  134. 134 
  135. 135 <p>To return an array of all POST items call without any parameters.</p>
  136. 136 <p>To return all POST items and pass them through the XSS filter set the first parameter NULL while setting the second parameter to boolean;</p>
  137. 137 <p>The function returns FALSE (boolean) if there are no items in the POST.</p>
  138. 138 
  139. 139 <code>
  140. 140  $this->input->post(NULL, TRUE); // returns all POST items with XSS filter
  141. 141  <br />
  142. 142  $this->input->post(); // returns all POST items without XSS filter
  143. 143 </code>
  144. 144 
  145. 145 <h2>$this->input->get()</h2>
  146. 146 
  147. 147 <p>This function is identical to the post function, only it fetches get data:</p>
  148. 148 
  149. 149 <code>$this->input->get('some_data', TRUE);</code>
  150. 150 
  151. 151 <p>To return an array of all GET items call without any parameters.</p>
  152. 152 <p>To return all GET items and pass them through the XSS filter set the first parameter NULL while setting the second parameter to boolean;</p>
  153. 153 <p>The function returns FALSE (boolean) if there are no items in the GET.</p>
  154. 154 
  155. 155 <code>
  156. 156  $this->input->get(NULL, TRUE); // returns all GET items with XSS filter
  157. 157  <br />
  158. 158  $this->input->get(); // returns all GET items without XSS filtering
  159. 159 </code>
  160. 160 
  161. 161 <h2>$this->input->get_post()</h2>
  162. 162 
  163. 163 <p>This function will search through both the post and get streams for data, looking first in post, and then in get:</p>
  164. 164 
  165. 165 <code>$this->input->get_post('some_data', TRUE);</code>
  166. 166 
  167. 167 <h2>$this->input->cookie()</h2>
  168. 168 
  169. 169 <p>This function is identical to the post function, only it fetches cookie data:</p>
  170. 170 
  171. 171 <code>$this->input->cookie('some_data', TRUE);</code>
  172. 172 
  173. 173 <h2>$this->input->server()</h2>
  174. 174 
  175. 175 <p>This function is identical to the above functions, only it fetches server data:</p>
  176. 176 
  177. 177 <code>$this->input->server('some_data');</code>
  178. 178 
  179. 179 
  180. 180 <h2>$this->input->set_cookie()</h2>
  181. 181 
  182. 182 <p>Sets a cookie containing the values you specify.  There are two ways to pass information to this function so that a cookie can be set:
  183. 183 Array Method, and Discrete Parameters:</p>
  184. 184 
  185. 185 <h4>Array Method</h4>
  186. 186 
  187. 187 <p>Using this method, an associative array is passed to the first parameter:</p>
  188. 188 
  189. 189 <code>$cookie = array(<br />
  190. 190 &nbsp;&nbsp;&nbsp;&nbsp;'name'&nbsp;&nbsp;&nbsp;=> 'The Cookie Name',<br />
  191. 191 &nbsp;&nbsp;&nbsp;&nbsp;'value'&nbsp;&nbsp;=> 'The Value',<br />
  192. 192 &nbsp;&nbsp;&nbsp;&nbsp;'expire' => '86500',<br />
  193. 193 &nbsp;&nbsp;&nbsp;&nbsp;'domain' => '.some-domain.com',<br />
  194. 194 &nbsp;&nbsp;&nbsp;&nbsp;'path'&nbsp;&nbsp;&nbsp;=> '/',<br />
  195. 195 &nbsp;&nbsp;&nbsp;&nbsp;'prefix' => 'myprefix_',<br />
  196. 196 &nbsp;&nbsp;&nbsp;&nbsp;'secure' => TRUE<br />
  197. 197 );<br />
  198. 198 <br />
  199. 199 $this->input->set_cookie($cookie);
  200. 200 </code>
  201. 201 
  202. 202 <p><strong>Notes:</strong></p>
  203. 203 
  204. 204 <p>Only the name and value are required. To delete a cookie set it with the expiration blank.</p>
  205. 205 
  206. 206 <p>The expiration is set in <strong>seconds</strong>, which will be added to the current time.  Do not include the time, but rather only the
  207. 207 number of seconds from <em>now</em> that you wish the cookie to be valid.  If the expiration is set to
  208. 208 zero the cookie will only last as long as the browser is open.</p>
  209. 209 <p>For site-wide cookies regardless of how your site is requested, add your URL to the <strong>domain</strong> starting with a period, like this:  .your-domain.com</p>
  210. 210 <p>The path is usually not needed since the function sets a root path.</p>
  211. 211 <p>The prefix is only needed if you need to avoid name collisions with other identically named cookies for your server.</p>
  212. 212 <p>The secure boolean is only needed if you want to make it a secure cookie by setting it to TRUE.</p>
  213. 213 
  214. 214 <h4>Discrete Parameters</h4>
  215. 215 
  216. 216 <p>If you prefer, you can set the cookie by passing data using individual parameters:</p>
  217. 217 
  218. 218 <code>$this->input->set_cookie($name, $value, $expire, $domain, $path, $prefix, $secure);</code>
  219. 219 
  220. 220 <h2>$this->input->cookie()</h2>
  221. 221 
  222. 222 <p>Lets you fetch a cookie.  The first parameter will contain the name of the cookie you are looking for (including any prefixes):</p>
  223. 223 
  224. 224 <code>cookie('some_cookie');</code>
  225. 225 
  226. 226 <p>The function returns FALSE (boolean) if the item you are attempting to retrieve does not exist.</p>
  227. 227 
  228. 228 <p>The second optional parameter lets you run the data through the XSS filter.  It's enabled by setting the second parameter to boolean TRUE;</p>
  229. 229 
  230. 230 <p><code>cookie('some_cookie', TRUE);</code></p>
  231. 231 
  232. 232 
  233. 233 <h2>$this->input->ip_address()</h2>
  234. 234 <p>Returns the IP address for the current user.  If the IP address is not valid, the function will return an IP of: 0.0.0.0</p>
  235. 235 <code>echo $this->input->ip_address();</code>
  236. 236 
  237. 237 
  238. 238 <h2>$this->input->valid_ip(<var>$ip</var>)</h2>
  239. 239 
  240. 240 <p>Takes an IP address as input and returns TRUE or FALSE (boolean) if it is valid or not.  Note:  The $this->input->ip_address() function above
  241. 241 validates the IP automatically.</p>
  242. 242 
  243. 243 <code>if ( ! $this-&gt;input-&gt;valid_ip($ip))<br />
  244. 244 {<br />
  245. 245 &nbsp;&nbsp;&nbsp;&nbsp; echo 'Not Valid';<br />
  246. 246 }<br />
  247. 247 else<br />
  248. 248 {<br />
  249. 249 &nbsp;&nbsp;&nbsp;&nbsp; echo 'Valid';<br />
  250. 250 }</code>
  251. 251 <p>Accepts an optional second string parameter of "IPv4" or "IPv6" to specify an IP format. The default checks for both formats.</p>
  252. 252 
  253. 253 <h2>$this->input->user_agent()</h2>
  254. 254 <p>Returns the user agent (web browser) being used by the current user. Returns FALSE if it's not available.</p>
  255. 255 <code>echo $this->input->user_agent();</code>
  256. 256 <p>See the <a href="user_agent.html">User Agent Class</a> for methods which extract information from the user agent string.</p>
  257. 257 
  258. 258 <h2>$this->input->request_headers()</h2>
  259. 259 <p>Useful if running in a non-Apache environment where <a href="http://php.net/apache_request_headers">apache_request_headers()</a> will not be supported.  Returns an array of headers.</p>
  260. 260 
  261. 261 <code>$headers = $this->input->request_headers();</code>
  262. 262 
  263. 263 <h2>$this->input->get_request_header();</h2>
  264. 264 <p>Returns a single member of the request headers array.</p>
  265. 265 
  266. 266 <code>$this->input->get_request_header('some-header', TRUE);</code>
  267. 267 
  268. 268 
  269. 269 <h2>$this->input->is_ajax_request()</h2>
  270. 270 <p>Checks to see if the <var>HTTP_X_REQUESTED_WITH</var> server header has been set, and returns a boolean response.</p>
  271. 271 
  272. 272 
  273. 273 <h2>$this->input->is_cli_request()</h2>
  274. 274 <p>Checks to see if the <var>STDIN</var> constant is set, which is a failsafe way to see if PHP is being run on the command line.</p>
  275. 275 
  276. 276 <code>$this->input->is_cli_request()</code>
