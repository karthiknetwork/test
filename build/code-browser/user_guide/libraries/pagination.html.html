  59.  59 <h1>Pagination Class</h1>
  60.  60 
  61.  61 <p>CodeIgniter's Pagination class is very easy to use, and it is 100% customizable, either dynamically or via stored preferences.</p>
  62.  62 
  63.  63 <p>If you are not familiar with the term "pagination", it refers to links that allows you to navigate from page to page, like this:</p>
  64.  64 
  65.  65 <code><a href="#">&laquo; First</a>&nbsp;&nbsp;<a href="#">&lt;</a>&nbsp;<a href="#">1</a>&nbsp;<a href="#">2</a>&nbsp;<b>3</b>&nbsp;<a href="#">4</a>&nbsp;<a href="#">5</a>&nbsp;<a href="#">&gt;</a>&nbsp;&nbsp;<a href="#">Last &raquo;</a></code>
  66.  66 
  67.  67 <h2>Example</h2>
  68.  68 
  69.  69 <p>Here is a simple example showing how to create pagination in one of your <a href="../general/controllers.html">controller</a> functions:</p>
  70.  70 
  71.  71 <code>
  72.  72 $this->load->library('pagination');<br /><br />
  73.  73 $config['base_url'] = 'http://example.com/index.php/test/page/';<br />
  74.  74 $config['total_rows'] = 200;<br />
  75.  75 $config['per_page']  = 20;
  76.  76 <br /><br />
  77.  77 $this->pagination->initialize($config);
  78.  78 
  79.  79 <br /><br />
  80.  80 echo  $this->pagination->create_links();</code>
  81.  81 
  82.  82 <h3>Notes:</h3>
  83.  83 
  84.  84 <p>The <var>$config</var> array contains your configuration variables.  It is passed to the <dfn>$this->pagination->initialize</dfn> function as shown above.  Although there are some twenty items you can configure, at
  85.  85 minimum you need the three shown.  Here is a description of what those items represent:</p>
  86.  86 
  87.  87 <ul>
  88.  88  <li><strong>base_url</strong> This is the full URL to the controller class/function containing your pagination.  In the example
  89.  89  above, it is pointing to a controller called "Test" and a function called "page".  Keep in mind that you can
  90.  90  <a href="../general/routing.html">re-route your URI</a> if you need a different structure.</li>
  91.  91  <li><strong>total_rows</strong> This number represents the total rows in the result set you are creating pagination for.
  92.  92  Typically this number will be the total rows that your database query returned.
  93.  93  </li>
  94.  94  <li><strong>per_page</strong> The number of items you intend to show per page.  In the above example, you would be showing 20 items per page.</li>
  95.  95 </ul>
  96.  96 
  97.  97 <p>The <var>create_links()</var> function returns an empty string when there is no pagination to show.</p>
  98.  98 
  99.  99 
  100. 100 <h3>Setting preferences in a config file</h3>
  101. 101 
  102. 102 <p>If you prefer not to set preferences using the above method, you can instead put them into a config file.
  103. 103 Simply create a new file called <var>pagination.php</var>,  add the <var>$config</var>
  104. 104 array in that file. Then save the file in: <var>config/pagination.php</var> and it will be used automatically. You
  105. 105 will NOT need to use the <dfn>$this->pagination->initialize</dfn> function if you save your preferences in a config file.</p>
  106. 106 
  107. 107 
  108. 108 <h2>Customizing the Pagination</h2>
  109. 109 
  110. 110 <p>The following is a list of all the preferences you can pass to the initialization function to tailor the display.</p>
  111. 111 
  112. 112 
  113. 113 <h4>$config['uri_segment'] = 3;</h4>
  114. 114 
  115. 115 <p>The pagination function automatically determines which segment of your URI contains the page number. If you need
  116. 116 something different you can specify it.</p>
  117. 117 
  118. 118 <h4>$config['num_links'] = 2;</h4>
  119. 119 
  120. 120 <p>The number of &quot;digit&quot; links you would like before and after the selected page number. For example, the number 2
  121. 121  will place two digits on either side, as in the example links at the very top of this page.</p>
  122. 122 
  123. 123 <h4>$config['use_page_numbers'] = TRUE;</h4>
  124. 124 <p>By default, the URI segment will use the starting index for the items you are paginating. If you prefer to show the the actual page number, set this to TRUE.</p>
  125. 125 
  126. 126 <h4>$config['page_query_string'] = TRUE;</h4>
  127. 127 <p>By default, the pagination library assume you are using <a href="../general/urls.html">URI Segments</a>, and constructs your links something like</p>
  128. 128 <p><code>http://example.com/index.php/test/page/20</code></p>
  129. 129 <p>If you have $config['enable_query_strings']  set to TRUE your links will automatically be re-written using Query Strings. This option can also be explictly set. Using $config['page_query_string'] set to TRUE, the pagination link will become.</p>
  130. 130 <p><code>http://example.com/index.php?c=test&amp;m=page&amp;per_page=20</code></p>
  131. 131 <p>Note that &quot;per_page&quot; is the default query string  passed, however can be configured using $config['query_string_segment'] = 'your_string'</p>
  132. 132 <h2>Adding Enclosing Markup</h2>
  133. 133 
  134. 134 <p>If you would like to surround the entire pagination with some markup you can do it with these two prefs:</p>
  135. 135 
  136. 136 <h4>$config['full_tag_open'] = '&lt;p>';</h4>
  137. 137 <p>The opening tag placed on the left side of the entire result.</p>
  138. 138 
  139. 139 <h4>$config['full_tag_close'] = '&lt;/p>';</h4>
  140. 140 <p>The closing tag placed on the right side of the entire result.</p>
  141. 141 
  142. 142 
  143. 143 <h2>Customizing the First Link</h2>
  144. 144 
  145. 145 <h4>$config['first_link'] = 'First';</h4>
  146. 146 <p>The text you would like shown in the "first" link on the left. If you do not want this link rendered, you can set its value to FALSE.</p>
  147. 147 
  148. 148 <h4>$config['first_tag_open'] = '&lt;div>';</h4>
  149. 149 <p>The opening tag for the "first" link.</p>
  150. 150 
  151. 151 <h4>$config['first_tag_close'] = '&lt;/div>';</h4>
  152. 152 <p>The closing tag for the "first" link.</p>
  153. 153 
  154. 154 <h2>Customizing the Last Link</h2>
  155. 155 
  156. 156 <h4>$config['last_link'] = 'Last';</h4>
  157. 157 <p>The text you would like shown in the "last" link on the right. If you do not want this link rendered, you can set its value to FALSE.</p>
  158. 158 
  159. 159 <h4>$config['last_tag_open'] = '&lt;div>';</h4>
  160. 160 <p>The opening tag for the "last" link.</p>
  161. 161 
  162. 162 <h4>$config['last_tag_close'] = '&lt;/div>';</h4>
  163. 163 <p>The closing tag for the "last" link.</p>
  164. 164 
  165. 165 <h2>Customizing the "Next" Link</h2>
  166. 166 
  167. 167 <h4>$config['next_link'] = '&amp;gt;';</h4>
  168. 168 <p>The text you would like shown in the "next" page link. If you do not want this link rendered, you can set its value to FALSE.</p>
  169. 169 
  170. 170 <h4>$config['next_tag_open'] = '&lt;div>';</h4>
  171. 171 <p>The opening tag for the "next" link.</p>
  172. 172 
  173. 173 <h4>$config['next_tag_close'] = '&lt;/div>';</h4>
  174. 174 <p>The closing tag for the "next" link.</p>
  175. 175 
  176. 176 <h2>Customizing the "Previous" Link</h2>
  177. 177 
  178. 178 <h4>$config['prev_link'] = '&amp;lt;';</h4>
  179. 179 <p>The text you would like shown in the "previous" page link. If you do not want this link rendered, you can set its value to FALSE.</p>
  180. 180 
  181. 181 <h4>$config['prev_tag_open'] = '&lt;div>';</h4>
  182. 182 <p>The opening tag for the "previous" link.</p>
  183. 183 
  184. 184 <h4>$config['prev_tag_close'] = '&lt;/div>';</h4>
  185. 185 <p>The closing tag for the "previous" link.</p>
  186. 186 
  187. 187 <h2>Customizing the "Current Page" Link</h2>
  188. 188 
  189. 189 <h4>$config['cur_tag_open'] = '&lt;b>';</h4>
  190. 190 <p>The opening tag for the "current" link.</p>
  191. 191 
  192. 192 <h4>$config['cur_tag_close'] = '&lt;/b>';</h4>
  193. 193 <p>The closing tag for the "current" link.</p>
  194. 194 
  195. 195 
  196. 196 <h2>Customizing the "Digit" Link</h2>
  197. 197 
  198. 198 <h4>$config['num_tag_open'] = '&lt;div>';</h4>
  199. 199 <p>The opening tag for the "digit" link.</p>
  200. 200 
  201. 201 <h4>$config['num_tag_close'] = '&lt;/div>';</h4>
  202. 202 <p>The closing tag for the "digit" link.</p>
  203. 203 
  204. 204 <h2>Hiding the Pages</h2>
  205. 205 
  206. 206 <p>If you wanted to not list the specific pages (for example, you only want "next" and "previous" links), you can suppress their rendering by adding:</p>
  207. 207 
  208. 208 <code>
  209. 209 $config['display_pages'] = FALSE;
  210. 210 </code>
  211. 211 
  212. 212 
  213. 213 <h2>Adding a class to every anchor</h2>
  214. 214 
  215. 215 <p>If you want to add a class attribute to every link rendered by the pagination class, you can set the config "anchor_class" equal to the classname you want.</p>
