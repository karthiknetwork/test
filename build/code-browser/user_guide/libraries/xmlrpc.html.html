  59.  59 <h1>XML-RPC and XML-RPC Server Classes</h1>
  60.  60 
  61.  61 
  62.  62 <p>CodeIgniter's XML-RPC classes permit you to send requests to another server, or set up
  63.  63 your own XML-RPC server to receive requests.</p>
  64.  64 
  65.  65 
  66.  66 <h2>What is XML-RPC?</h2>
  67.  67 
  68.  68 <p>Quite simply it is a way for two computers to communicate over the internet using XML.
  69.  69 One computer, which we will call the <dfn>client</dfn>, sends an XML-RPC <strong>request</strong> to
  70.  70 another computer, which we will call the <dfn>server</dfn>.  Once the server receives and processes the request it
  71.  71 will send back a <strong>response</strong> to the client.</p>
  72.  72 
  73.  73 <p>For example, using the MetaWeblog API, an XML-RPC Client (usually a desktop publishing tool) will
  74.  74 send a request to an XML-RPC Server running on your site.  This request might be a new weblog entry
  75.  75 being sent for publication, or it could be a request for an existing entry for editing.
  76.  76 
  77.  77 When the XML-RPC Server receives this request it will examine it to determine which class/method should be called to process the request.
  78.  78 Once processed, the server will then send back a response message.</p>
  79.  79 
  80.  80 <p>For detailed specifications, you can visit the <a href="http://www.xmlrpc.com/">XML-RPC</a> site.</p>
  81.  81 
  82.  82 <h2>Initializing the Class</h2>
  83.  83 
  84.  84 <p>Like most other classes in CodeIgniter, the XML-RPC and XML-RPCS classes are initialized in your controller using the <dfn>$this->load->library</dfn> function:</p>
  85.  85 
  86.  86 <p>To load the XML-RPC class you will use:</p>
  87.  87 <code>$this->load->library('xmlrpc');</code>
  88.  88 <p>Once loaded, the xml-rpc library object will be available using: <dfn>$this->xmlrpc</dfn></p>
  89.  89 
  90.  90 <p>To load the XML-RPC Server class you will use:</p>
  91.  91 <code>
  92.  92 $this->load->library('xmlrpc');<br />
  93.  93 $this->load->library('xmlrpcs');
  94.  94 </code>
  95.  95 <p>Once loaded, the xml-rpcs library object will be available using: <dfn>$this->xmlrpcs</dfn></p>
  96.  96 
  97.  97 <p class="important"><strong>Note:</strong>&nbsp; When using the XML-RPC Server class you must load BOTH the XML-RPC class and the XML-RPC Server class.</p>
  98.  98 
  99.  99 
  100. 100 
  101. 101 <h2>Sending XML-RPC Requests</h2>
  102. 102 
  103. 103 <p>To send a request to an XML-RPC server you must specify the following information:</p>
  104. 104 
  105. 105 <ul>
  106. 106 <li>The URL of the server</li>
  107. 107 <li>The method on the server you wish to call</li>
  108. 108 <li>The <em>request</em> data (explained below).</li>
  109. 109 </ul>
  110. 110 
  111. 111 <p>Here is a basic example that sends a simple Weblogs.com ping to the <a href="http://pingomatic.com/">Ping-o-Matic</a></p>
  112. 112 
  113. 113 
  114. 114 <code>$this->load->library('xmlrpc');<br />
  115. 115 <br />
  116. 116 $this->xmlrpc->server('http://rpc.pingomatic.com/', 80);<br />
  117. 117 $this->xmlrpc->method('weblogUpdates.ping');<br />
  118. 118 
  119. 119 <br />
  120. 120 $request = array('My Photoblog', 'http://www.my-site.com/photoblog/');<br />
  121. 121 $this->xmlrpc->request($request);<br />
  122. 122 <br />
  123. 123 if ( ! $this->xmlrpc->send_request())<br />
  124. 124 {<br />
  125. 125 &nbsp;&nbsp;&nbsp;&nbsp;echo $this-&gt;xmlrpc->display_error();<br />
  126. 126 }</code>
  127. 127 
  128. 128 <h3>Explanation</h3>
  129. 129 
  130. 130 <p>The above code initializes the XML-RPC class, sets the server URL and method to be called (weblogUpdates.ping).  The
  131. 131 request (in this case, the title and URL of your site) is placed into an array for transportation, and
  132. 132 compiled using the request() function.
  133. 133 Lastly, the full request is sent.  If the <dfn>send_request()</dfn> method returns false we will display the error message
  134. 134 sent back from the XML-RPC Server.</p>
  135. 135 
  136. 136 <h2>Anatomy of a Request</h2>
  137. 137 
  138. 138 <p>An XML-RPC <dfn>request</dfn> is simply the data you are sending to the XML-RPC server.  Each piece of data in a request
  139. 139 is referred to as a <dfn>request parameter</dfn>.  The above example has two parameters:
  140. 140 The URL and title of your site. When the XML-RPC server receives your request, it will look for parameters it requires.</p>
  141. 141 
  142. 142 <p>Request parameters must be placed into an array for transportation, and each parameter can be one
  143. 143 of seven data types (strings, numbers, dates, etc.). If your parameters are something other than strings
  144. 144 you will have to include the data type in the request array.</p>
  145. 145 
  146. 146 <p>Here is an example of a simple array with three parameters:</p>
  147. 147 
  148. 148 <code>$request = array('John', 'Doe', 'www.some-site.com');<br />
  149. 149 $this->xmlrpc->request($request);</code>
  150. 150 
  151. 151 <p>If you use data types other than strings, or if you have several different data types, you will place
  152. 152 each parameter into its own array, with the data type in the second position:</p>
  153. 153 
  154. 154 <code>
  155. 155 $request = array (<br />
  156. 156 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array('John', 'string'),<br />
  157. 157 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array('Doe', 'string'),<br />
  158. 158 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array(FALSE, 'boolean'),<br />
  159. 159 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array(12345, 'int')<br />
  160. 160 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;);
  161. 161 <br />
  162. 162 $this->xmlrpc->request($request);</code>
  163. 163 
  164. 164 The  <a href="#datatypes">Data Types</a>  section below has a full list of data types.
  165. 165 
  166. 166 
  167. 167 
  168. 168 <h2>Creating an XML-RPC Server</h2>
  169. 169 
  170. 170 <p>An XML-RPC Server acts as a traffic cop of sorts, waiting for incoming requests and redirecting them to the
  171. 171 appropriate functions for processing.</p>
  172. 172 
  173. 173 <p>To create your own XML-RPC server involves initializing the XML-RPC Server class in your controller where you expect the incoming
  174. 174 request to appear, then setting up an array with mapping instructions so that incoming requests can be sent to the appropriate
  175. 175 class and method for processing.</p>
  176. 176 
  177. 177 <p>Here is an example to illustrate:</p>
  178. 178 
  179. 179 <code>
  180. 180 $this->load->library('xmlrpc');<br />
  181. 181 $this->load->library('xmlrpcs');<br />
  182. 182 <br />
  183. 183 $config['functions']['<var>new_post</var>'] = array('function' => '<dfn>My_blog.new_entry</dfn>'),<br />
  184. 184 $config['functions']['<var>update_post</var>'] = array('function' => '<dfn>My_blog.update_entry</dfn>');<br />
  185. 185 $config['object'] = $this;<br />
  186. 186 <br />
  187. 187 $this->xmlrpcs->initialize($config);<br />
  188. 188 $this->xmlrpcs->serve();</code>
  189. 189 
  190. 190 <p>The above example contains an array specifying two method requests that the Server allows.
  191. 191 The allowed methods are on the left side of the array. When either of those are received, they will be mapped to the class and method on the right.</p>
  192. 192 
  193. 193 <p>The '<var>object</var>' key is a special key that you pass an instantiated class object with, which is necessary when the method you are mapping to is not
  194. 194  part of the CodeIgniter super object.</p>
  195. 195 
  196. 196 <p>In other words, if an XML-RPC Client sends a request for the <var>new_post</var> method, your
  197. 197 server will load the <dfn>My_blog</dfn> class and call the <dfn>new_entry</dfn> function.
  198. 198 If the request is for the <var>update_post</var> method, your
  199. 199 server will load the <dfn>My_blog</dfn> class and call the <dfn>update_entry</dfn> function.</p>
  200. 200 
  201. 201 <p>The function names in the above example are arbitrary.  You'll decide what they should be called on your server,
  202. 202 or if you are using standardized APIs, like the Blogger or MetaWeblog API, you'll use their function names.</p>
  203. 203 
  204. 204 <p>There are two additional configuration keys you may make use of when initializing the server class: <var>debug</var> can be set to TRUE in order to enable debugging, and <var>xss_clean</var> may be set to FALSE to prevent sending data through the Security library's xss_clean function.
  205. 205 
  206. 206 <h2>Processing Server Requests</h2>
  207. 207 
  208. 208 <p>When the XML-RPC Server receives a request and loads the class/method for processing, it will pass
  209. 209 an object to that method containing the data sent by the client.</p>
  210. 210 
  211. 211 <p>Using the above example, if the <var>new_post</var> method is requested, the server will expect a class
  212. 212 to exist with this prototype:</p>
  213. 213 
  214. 214 <code>class <kbd>My_blog</kbd> extends CI_Controller {<br />
  215. 215 <br />
  216. 216 &nbsp;&nbsp;&nbsp;&nbsp;function <kbd>new_post</kbd>(<var>$request</var>)<br />
  217. 217 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  218. 218  <br />
  219. 219 &nbsp;&nbsp;&nbsp;&nbsp;}<br />
  220. 220 }
  221. 221 </code>
  222. 222 
  223. 223 <p>The <var>$request</var> variable is an object compiled by the Server, which contains the data sent by the XML-RPC Client.
  224. 224 Using this object you will have access to the <em>request parameters</em> enabling you to process the request.  When
  225. 225 you are done you will send a <dfn>Response</dfn> back to the Client.</p>
  226. 226 
  227. 227 <p>Below is a real-world example, using the Blogger API.  One of the methods in the Blogger API is <dfn>getUserInfo()</dfn>.
  228. 228 Using this method, an XML-RPC Client can send the Server a username and password, in return the Server sends
  229. 229 back information about that particular user (nickname, user ID, email address, etc.).  Here is how the processing
  230. 230 function might look:</p>
  231. 231 
  232. 232 
  233. 233 <code>class <kbd>My_blog</kbd> extends CI_Controller {<br />
  234. 234 <br />
  235. 235 &nbsp;&nbsp;&nbsp;&nbsp;function <kbd>getUserInfo</kbd>(<var>$request</var>)<br />
  236. 236 &nbsp;&nbsp;&nbsp;&nbsp;{<br />
  237. 237 
  238. 238 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$username = 'smitty';<br />
  239. 239 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$password = 'secretsmittypass';<br /><br />
  240. 240 
  241. 241 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$this->load->library('xmlrpc');<br />
  242. 242 &nbsp;&nbsp;&nbsp;&nbsp;<br />
  243. 243 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$parameters&nbsp;=&nbsp;$request->output_parameters();<br />
  244. 244 &nbsp;&nbsp;&nbsp;&nbsp;<br />
  245. 245 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;if&nbsp;($parameters['1'] != $username AND $parameters['2'] != $password)<br />
  246. 246 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;{<br />
  247. 247 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return&nbsp;$this->xmlrpc->send_error_message('100',&nbsp;'Invalid&nbsp;Access');<br />
  248. 248 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;}<br />
  249. 249 &nbsp;&nbsp;&nbsp;&nbsp;<br />
  250. 250 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;$response&nbsp;=&nbsp;array(array('nickname'&nbsp;&nbsp;=>&nbsp;array('Smitty','string'),<br />
  251. 251 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'userid'&nbsp;&nbsp;&nbsp;&nbsp;=>&nbsp;array('99','string'),<br />
  252. 252 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'url'&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;=>&nbsp;array('http://yoursite.com','string'),<br />
  253. 253 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'email'&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;=>&nbsp;array('jsmith@yoursite.com','string'),<br />
  254. 254 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'lastname'&nbsp;&nbsp;=>&nbsp;array('Smith','string'),<br />
  255. 255 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'firstname'&nbsp;=>&nbsp;array('John','string')<br />
  256. 256 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;),<br />
  257. 257 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'struct');<br />
  258. 258 <br />
  259. 259 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;return $this->xmlrpc->send_response($response);<br />
  260. 260 &nbsp;&nbsp;&nbsp;&nbsp;}<br />
  261. 261 }
  262. 262 </code>
  263. 263 
  264. 264 <h3>Notes:</h3>
  265. 265 <p>The <dfn>output_parameters()</dfn> function retrieves an indexed array corresponding to the request parameters sent by the client.
  266. 266 In the above example, the output parameters will be the username and password.</p>
  267. 267 
  268. 268 <p>If the username and password sent by the client were not valid, and error message is returned using <dfn>send_error_message()</dfn>.</p>
  269. 269 
  270. 270 <p>If the operation was successful, the client will be sent back a response array containing the user's info.</p>
  271. 271 
  272. 272 
  273. 273 <h2>Formatting a Response</h2>
  274. 274 
  275. 275 <p>Similar to <em>Requests</em>, <em>Responses</em> must be formatted as an array.  However, unlike requests, a response is an array
  276. 276 <strong>that contains a single item</strong>.  This item can be an array with several additional arrays, but there
  277. 277 can be only one primary array index.  In other words, the basic prototype is this:</p>
  278. 278 
  279. 279 <code>$response = array('Response data',  'array');</code>
  280. 280 
  281. 281 <p>Responses, however, usually contain multiple pieces of information. In order to accomplish this we must put the response into its own
  282. 282 array so that the primary array continues to contain a single piece of data.  Here's an example showing how this might be accomplished:</p>
  283. 283 
  284. 284 <code>
  285. 285 $response = array (<br />
  286. 286 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array(<br />

  314. 314 <h2>Creating Your Own Client and Server</h2>
  315. 315 
  316. 316 <p>To help you understand everything we've covered thus far, let's create a couple controllers that act as
  317. 317 XML-RPC Client and Server.  You'll use the Client to send a request to the Server and receive a response.</p>
  318. 318 
  319. 319 <h3>The Client</h3>
  320. 320 
  321. 321 <p>Using a text editor, create a controller called <dfn>xmlrpc_client.php</dfn>.
  322. 322 In it, place this code and save it to your <samp>applications/controllers/</samp> folder:</p>
  323. 323 
  324. 324 <textarea class="textarea" style="width:100%" cols="50" rows="32">&lt;?php
  325. 325 
  326. 326 class Xmlrpc_client extends CI_Controller {
  327. 327 
  328. 328  function index()
  329. 329  {
  330. 330  $this->load->helper('url');
  331. 331  $server_url = site_url('xmlrpc_server');
  332. 332 
  333. 333  $this->load->library('xmlrpc');
  334. 334 
  335. 335  $this->xmlrpc->server($server_url, 80);
  336. 336  $this->xmlrpc->method('Greetings');
  337. 337 
  338. 338  $request = array('How is it going?');
  339. 339  $this->xmlrpc->request($request);
  340. 340 
  341. 341  if ( ! $this->xmlrpc->send_request())
  342. 342  {
  343. 343  echo $this->xmlrpc->display_error();
  344. 344  }
  345. 345  else
  346. 346  {
  347. 347  echo '<pre>';
  348. 348  print_r($this->xmlrpc->display_response());
  349. 349  echo '</pre>';
  350. 350  }
  351. 351  }
  352. 352 }
  353. 353 ?></textarea>
  354. 354 
  355. 355 <p>Note: In the above code we are using a "url helper".  You can find more information in the <a href="../general/helpers.html">Helpers Functions</a> page.</p>
  356. 356 
  357. 357 <h3>The Server</h3>
  358. 358 
  359. 359 <p>Using a text editor, create a controller called <dfn>xmlrpc_server.php</dfn>.
  360. 360 In it, place this code and save it to your <samp>applications/controllers/</samp> folder:</p>
  361. 361 
  362. 362 <textarea class="textarea" style="width:100%" cols="50" rows="30">&lt;?php
  363. 363 
  364. 364 class Xmlrpc_server extends CI_Controller {
  365. 365 
  366. 366  function index()
  367. 367  {
  368. 368  $this->load->library('xmlrpc');
  369. 369  $this->load->library('xmlrpcs');
  370. 370 
  371. 371  $config['functions']['Greetings'] = array('function' => 'Xmlrpc_server.process');
  372. 372 
  373. 373  $this->xmlrpcs->initialize($config);
  374. 374  $this->xmlrpcs->serve();
  375. 375  }
  376. 376 
  377. 377 
  378. 378  function process($request)
  379. 379  {
  380. 380  $parameters = $request->output_parameters();
  381. 381 
  382. 382  $response = array(
  383. 383  array(
  384. 384  'you_said'  => $parameters['0'],
  385. 385  'i_respond' => 'Not bad at all.'),
  386. 386  'struct');
  387. 387 
  388. 388  return $this->xmlrpc->send_response($response);
  389. 389  }
  390. 390 }
  391. 391 ?></textarea>
  392. 392 
  393. 393 <h3>Try it!</h3>
  394. 394 
  395. 395 <p>Now visit the your site using a URL similar to this:</p>
  396. 396 <code>example.com/index.php/<var>xmlrpc_client</var>/</code>
  397. 397 
  398. 398 <p>You should now see the message you sent to the server, and its response back to you.</p>
  399. 399 
  400. 400 <p>The client you created sends a message ("How's is going?") to the server, along with a request for the "Greetings" method.
  401. 401 The Server receives the request and maps it to the "process" function, where a response is sent back.</p>

  433. 433 <h1>XML-RPC Function Reference</h1>
  434. 434 
  435. 435 <h2>$this->xmlrpc->server()</h2>
  436. 436 <p>Sets the URL and port number of the server to which a request is to be sent:</p>
  437. 437 <code>$this->xmlrpc->server('http://www.sometimes.com/pings.php', 80);</code>
  438. 438 
  439. 439 <h2>$this->xmlrpc->timeout()</h2>
  440. 440 <p>Set a time out period (in seconds) after which the request will be canceled:</p>
  441. 441 <code>$this->xmlrpc->timeout(6);</code>
  442. 442 
  443. 443 <h2>$this->xmlrpc->method()</h2>
  444. 444 <p>Sets the method that will be requested from the XML-RPC server:</p>
  445. 445 <code>$this->xmlrpc->method('<var>method</var>');</code>
  446. 446 
  447. 447 <p>Where <var>method</var> is the name of the method.</p>
  448. 448 
  449. 449 <h2>$this->xmlrpc->request()</h2>
  450. 450 <p>Takes an array of data and builds request to be sent to XML-RPC server:</p>
  451. 451 <code>$request = array(array('My Photoblog', 'string'), 'http://www.yoursite.com/photoblog/');<br />
  452. 452 $this->xmlrpc->request($request);</code>
  453. 453 
  454. 454 <h2>$this->xmlrpc->send_request()</h2>
  455. 455 <p>The request sending function.  Returns boolean TRUE or FALSE based on success for failure, enabling it to be used conditionally.</p>
  456. 456 
  457. 457 <h2>$this->xmlrpc->set_debug(TRUE);</h2>
  458. 458 <p>Enables debugging, which will display a variety of information and error data helpful during development.</p>
  459. 459 
  460. 460 
  461. 461 <h2>$this->xmlrpc->display_error()</h2>
  462. 462 <p>Returns an error message as a string if your request failed for some reason.</p>
  463. 463 <code>echo $this->xmlrpc->display_error();</code>
  464. 464 
  465. 465 <h2>$this->xmlrpc->display_response()</h2>
  466. 466 <p>Returns the response from the remote server once request is received.  The response will typically be an associative array.</p>
  467. 467 <code>$this->xmlrpc->display_response();</code>
  468. 468 
  469. 469 <h2>$this->xmlrpc->send_error_message()</h2>
  470. 470 <p>This function lets you send an error message from your server to the client. First parameter is the error number while the second parameter
  471. 471 is the error message.</p>
  472. 472 <code>return $this->xmlrpc->send_error_message('123', 'Requested data not available');</code>
  473. 473 
  474. 474 <h2>$this->xmlrpc->send_response()</h2>
  475. 475 <p>Lets you send the response from your server to the client. An array of valid data values must be sent with this method.</p>
  476. 476 <code>$response = array(<br />
  477. 477 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;array(<br />
  478. 478 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'flerror' => array(FALSE, 'boolean'),<br />
  479. 479 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'message' => "Thanks for the ping!"<br />
  480. 480 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;)<br />
  481. 481 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;'struct');<br />
  482. 482 return $this->xmlrpc->send_response($response);</code>

  486. 486 <h2>Data Types</h2>
  487. 487 
  488. 488 <p>According to the <a href="http://www.xmlrpc.com/spec">XML-RPC spec</a> there are seven types

  491. 491 <ul>

  493. 493 <li><em>boolean</em></li>
  494. 494 <li><em>string</em></li>
  495. 495 <li><em>double</em></li>
  496. 496 <li><em>dateTime.iso8601</em></li>
  497. 497 <li><em>base64</em></li>
  498. 498 <li><em>struct</em> (contains array of values)</li>
  499. 499 <li><em>array</em> (contains array of values)</li>
