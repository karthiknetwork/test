  58.  58 <h1>CodeIgniter at a Glance</h1>
  59.  59 
  60.  60 
  61.  61 <h2>CodeIgniter is an Application Framework</h2>
  62.  62 
  63.  63 <p>CodeIgniter is a toolkit for people who build web applications using PHP.  Its goal is to enable you to develop projects much faster than you could if you were writing code
  64.  64 from scratch, by providing a rich set of libraries for commonly needed tasks, as well as a simple interface and
  65.  65 logical structure to access these libraries. CodeIgniter lets you creatively focus on your project by
  66.  66 minimizing the amount of code needed for a given task.</p>
  67.  67 
  68.  68 <h2>CodeIgniter is Free</h2>
  69.  69 <p>CodeIgniter is licensed under an Apache/BSD-style open source license so you can use it however you please.
  70.  70 For more information please read the <a href="../license.html">license agreement</a>.</p>
  71.  71 
  72.  72 <h2>CodeIgniter is Light Weight</h2>
  73.  73 <p>Truly light weight.  The core system requires only a few very small libraries. This is in stark contrast to many frameworks that require significantly more resources.
  74.  74 Additional libraries are loaded dynamically upon request, based on your needs for a given process, so the base system
  75.  75 is very lean and quite fast.
  76.  76 </p>
  77.  77 
  78.  78 <h2>CodeIgniter is Fast</h2>
  79.  79 <p>Really fast. We challenge you to find a framework that has better performance than CodeIgniter.</p>
  80.  80 
  81.  81 
  82.  82 <h2>CodeIgniter Uses M-V-C</h2>
  83.  83 <p>CodeIgniter uses the Model-View-Controller approach, which allows great separation between logic and presentation.
  84.  84 This is particularly good for projects in which designers are working with your template files, as the code these file contain will be minimized. We describe MVC in more detail on its own page.</p>
  85.  85 
  86.  86 <h2>CodeIgniter Generates Clean URLs</h2>
  87.  87 <p>The URLs generated by CodeIgniter are clean and search-engine friendly.  Rather than using the standard "query string"
  88.  88 approach to URLs that is synonymous with dynamic systems, CodeIgniter uses a segment-based approach:</p>
  89.  89 
  90.  90 <code>example.com/<var>news</var>/<dfn>article</dfn>/<samp>345</samp></code>
  91.  91 
  92.  92 <p>Note: By default the index.php file is included in the URL but it can be removed using a simple .htaccess file.</p>
  93.  93 
  94.  94 <h2>CodeIgniter Packs a Punch</h2>
  95.  95 <p>CodeIgniter comes with  full-range of libraries that enable the most commonly needed web development tasks,
  96.  96 like accessing a database, sending email, validating form data, maintaining sessions, manipulating images, working with XML-RPC data and
  97.  97 much more.</p>
  98.  98 
  99.  99 <h2>CodeIgniter is Extensible</h2>
  100. 100 <p>The system can be easily extended through the use of your own libraries, helpers, or through class extensions or system hooks.</p>
  101. 101 
  102. 102 
  103. 103 <h2>CodeIgniter Does Not Require a Template Engine</h2>
  104. 104 <p>Although CodeIgniter <em>does</em> come with a simple template parser that can be optionally used, it does not force you to use one.
  105. 105 
  106. 106 Template engines simply can not match the performance of native PHP, and the syntax that must be learned to use a template
  107. 107 engine is usually only marginally easier than learning the basics of PHP.  Consider this block of PHP code:</p>
  108. 108 
  109. 109 <code>&lt;ul><br />
  110. 110 <br />
  111. 111 &lt;?php foreach ($addressbook as $name):?><br />
  112. 112 <br />
  113. 113 &lt;li>&lt;?=$name?>&lt;/li><br />
  114. 114 <br />
  115. 115 &lt;?php endforeach; ?><br />
  116. 116 <br />
  117. 117 &lt;/ul></code>
  118. 118 
  119. 119 <p>Contrast this with the pseudo-code used by a template engine:</p>
  120. 120 
  121. 121 <code>&lt;ul><br />
  122. 122 <br />
  123. 123 {foreach from=$addressbook item="name"}<br />
  124. 124 <br />
  125. 125 &lt;li>{$name}&lt;/li><br />
  126. 126 <br />
  127. 127 {/foreach}<br />
  128. 128 <br />
  129. 129 &lt;/ul></code>
  130. 130 
  131. 131 <p>Yes, the template engine example is a bit cleaner, but it comes at the price of performance, as the pseudo-code must be converted
  132. 132 back into PHP to run. Since one of our goals is <em>maximum performance</em>, we opted to not require the use of a template engine.</p>
  133. 133 
  134. 134 
  135. 135 <h2>CodeIgniter is Thoroughly Documented</h2>
  136. 136 <p>Programmers love to code and hate to write documentation.  We're no different, of course, but
  137. 137 since documentation is <strong>as important</strong> as the code itself,
  138. 138 we are committed to doing it. Our source code is extremely clean and well commented as well.</p>
  139. 139 
  140. 140 
  141. 141 <h2>CodeIgniter has a Friendly Community of Users</h2>
  142. 142 
  143. 143 <p>Our growing community of users can be seen actively participating in our <a href="http://codeigniter.com/forums/">Community Forums</a>.</p>
