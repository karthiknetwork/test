  60.  60 <h1>Tutorial &minus; News section</h1>
  61.  61 
  62.  62 <p>In the last section, we went over some basic concepts of the framework by writing a class that includes static pages. We cleaned up the URI by adding custom routing rules. Now it's time to introduce dynamic content and start using a database.</p>
  63.  63 
  64.  64 <h2>Setting up your model</h2>
  65.  65 
  66.  66 <p>Instead of writing database operations right in the controller, queries should be placed in a model, so they can easily be reused later. Models are the place where you retrieve, insert, and update information in your database or other data stores. They represent your data.</p>
  67.  67 
  68.  68 <p>Open up the <dfn>application/models</dfn> directory and create a new file called <dfn>news_model.php</dfn> and add the following code. Make sure you've configured your database properly as described <a href="../database/configuration.html">here</a>.</p>
  69.  69 
  70.  70 <pre>
  71.  71 &lt;?php
  72.  72 class News_model extends CI_Model {
  73.  73 
  74.  74  public function __construct()
  75.  75  {
  76.  76  $this->load->database();
  77.  77  }
  78.  78 }
  79.  79 </pre>
  80.  80 
  81.  81 <p>This code looks similar to the controller code that was used earlier. It creates a new model by extending CI_Model and loads the database library. This will make the database class available through the <var>$this->db</var> object.</p>
  82.  82 
  83.  83 <p>Before querying the database, a database schema has to be created. Connect to your database and run the SQL command below. Also add some seed records.</p>
  84.  84 
  85.  85 <pre>
  86.  86 CREATE TABLE news (
  87.  87  id int(11) NOT NULL AUTO_INCREMENT,
  88.  88  title varchar(128) NOT NULL,
  89.  89  slug varchar(128) NOT NULL,
  90.  90  text text NOT NULL,
  91.  91  PRIMARY KEY (id),
  92.  92  KEY slug (slug)
  93.  93 );
  94.  94 </pre>
  95.  95 
  96.  96 <p>Now that the database and a model have been set up, you'll need a method to get all of our posts from our database. To do this, the database abstraction layer that is included with CodeIgniter — <a href="../database/active_record.html">Active Record</a> — is used. This makes it possible to write your 'queries' once and make them work on <a href="../general/requirements.html">all supported database systems</a>. Add the following code to your model.</p>
  97.  97 
  98.  98 <pre>
  99.  99 public function get_news($slug = FALSE)
  100. 100 {
  101. 101  if ($slug === FALSE)
  102. 102  {
  103. 103  $query = $this->db->get('news');
  104. 104  return $query->result_array();
  105. 105  }
  106. 106 
  107. 107  $query = $this->db->get_where('news', array('slug' => $slug));
  108. 108  return $query->row_array();
  109. 109 }
  110. 110 </pre>
  111. 111 
  112. 112 <p>With this code you can perform two different queries. You can get all news records, or get a news item by its <a href="#" title="a URL friendly version of a string">slug</a>. You might have noticed that the <var>$slug</var> variable wasn't sanitized before running the query; Active Record does this for you.</p>
  113. 113 
  114. 114 <h2>Display the news</h2>
  115. 115 
  116. 116 <p>Now that the queries are written, the model should be tied to the views that are going to display the news items to the user. This could be done in our pages controller created earlier, but for the sake of clarity, a new "news" controller is defined. Create the new controller at <dfn>application/controllers/news.php</dfn>.</p>
  117. 117 
  118. 118 <pre>
  119. 119 &lt;?php
  120. 120 class News extends CI_Controller {
  121. 121 
  122. 122  public function __construct()
  123. 123  {
  124. 124  parent::__construct();
  125. 125  $this->load->model('news_model');
  126. 126  }
  127. 127 
  128. 128  public function index()
  129. 129  {
  130. 130  $data['news'] = $this->news_model->get_news();
  131. 131  }
  132. 132 
  133. 133  public function view($slug)
  134. 134  {
  135. 135  $data['news'] = $this->news_model->get_news($slug);
  136. 136  }
  137. 137 }
  138. 138 </pre>
  139. 139 
  140. 140 <p>Looking at the code, you may see some similarity with the files we created earlier. First, the "__construct" method: it calls the constructor of its parent class (<dfn>CI_Controller</dfn>) and loads the model, so it can be used in all other methods in this controller.</p>
  141. 141 
  142. 142 <p>Next, there are two methods to view all news items and one for a specific news item. You can see that the <var>$slug</var> variable is passed to the model's method in the second method. The model is using this slug to identify the news item to be returned.</p>
  143. 143 
  144. 144 <p>Now the data is retrieved by the controller through our model, but nothing is displayed yet. The next thing to do is passing this data to the views.</p>
  145. 145 
  146. 146 <pre>
  147. 147 public function index()
  148. 148 {
  149. 149  $data['news'] = $this->news_model->get_news();
  150. 150  $data['title'] = 'News archive';
  151. 151 
  152. 152  $this->load->view('templates/header', $data);
  153. 153  $this->load->view('news/index', $data);
  154. 154  $this->load->view('templates/footer');
  155. 155 }
  156. 156 </pre>
  157. 157 
  158. 158 <p>The code above gets all news records from the model and assigns it to a variable. The value for the title is also assigned to the <var>$data['title']</var> element and all data is passed to the views. You now need to create a view to render the news items. Create <dfn>application/views/news/index.php</dfn> and add the next piece of code.</p>
  159. 159 
  160. 160 <pre>
  161. 161 &lt;?php foreach ($news as $news_item): ?&gt;
  162. 162 
  163. 163     &lt;h2&gt;&lt;?php echo $news_item['title'] ?&gt;&lt;/h2&gt;
  164. 164     &lt;div id="main"&gt;
  165. 165         &lt;?php echo $news_item['text'] ?&gt;
  166. 166     &lt;/div&gt;
  167. 167     &lt;p&gt;&lt;a href="news/&lt;?php echo $news_item['slug'] ?&gt;"&gt;View article&lt;/a&gt;&lt;/p&gt;
  168. 168 
  169. 169 &lt;?php endforeach ?&gt;
  170. 170 </pre>
  171. 171 
  172. 172 <p>Here, each news item is looped and displayed to the user. You can see we wrote our template in PHP mixed with HTML. If you prefer to use a template language, you can use CodeIgniter's <a href="../libraries/parser.html">Template Parser</a> class or a third party parser.</p>
  173. 173 
  174. 174 <p>The news overview page is now done, but a page to display individual news items is still absent. The model created earlier is made in such way that it can easily be used for this functionality. You only need to add some code to the controller and create a new view. Go back to the news controller and add the following lines to the file.</p>
  175. 175 
  176. 176 <pre>
  177. 177 public function view($slug)
  178. 178 {
  179. 179  $data['news_item'] = $this->news_model->get_news($slug);
  180. 180 
  181. 181  if (empty($data['news_item']))
  182. 182  {
  183. 183  show_404();
  184. 184  }
  185. 185 
  186. 186  $data['title'] = $data['news_item']['title'];
  187. 187 
  188. 188  $this->load->view('templates/header', $data);
  189. 189  $this->load->view('news/view', $data);
  190. 190  $this->load->view('templates/footer');
  191. 191 }
  192. 192 </pre>
  193. 193 
  194. 194 <p>Instead of calling the <var>get_news()</var> method without a parameter, the <var>$slug</var> variable is passed, so it will return the specific news item. The only things left to do is create the corresponding view at <dfn>application/views/news/view.php</dfn>. Put the following code in this file.</p>
  195. 195 
  196. 196 <pre>
  197. 197 &lt;?php
  198. 198 echo '&lt;h2&gt;'.$news_item['title'].'&lt;/h2&gt;';
  199. 199 echo $news_item['text'];
  200. 200 </pre>
  201. 201 
  202. 202 <h2>Routing</h2>
  203. 203 <p>Because of the wildcard routing rule created earlier, you need need an extra route to view the controller that you just made. Modify your routing file (<dfn>application/config/routes.php</dfn>) so it looks as follows. This makes sure the requests reaches the news controller instead of going directly to the pages controller. The first line routes URI's with a slug to the view method in the news controller.</p>
  204. 204 
  205. 205 <pre>
  206. 206 $route['news/(:any)'] = 'news/view/$1';
  207. 207 $route['news'] = 'news';
  208. 208 $route['(:any)'] = 'pages/view/$1';
  209. 209 $route['default_controller'] = 'pages/view';
  210. 210 </pre>
  211. 211 
  212. 212 <p>Point your browser to your document root, followed by <dfn>index.php/news</dfn> and watch your news page.</p>
